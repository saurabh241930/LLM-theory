RAGAS Framework

RAGAS = Retrieval-Augmented Generation Assessment.

Uses LLM-as-a-Judge to score 4 key metrics:

  • Faithfulness: Is answer supported by context?
  • Answer Relevancy: Does answer address the question?
  • Context Recall: Did we retrieve ALL needed info?
  • Context Precision: Are retrieved chunks relevant?

Metric 1 & 2: Generation

Faithfulness: LLM Check. "Is this claim in the context?" Detects hallucinations.

Answer Relevancy: LLM Check. "Does this answer the question?" Detects off-topic rambling.

Comparison
Faithfulness
Context: "Founded 1998"
Answer: "CEO is John"
Result: ❌ Hallucination
Answer Relevancy
Query: "What do they do?"
Answer: "Founded in 1950"
Result: ❌ Off-topic

Metric 3 & 4: Retrieval

Context Recall: Did we miss any necessary pieces of info?

Context Precision: Did we retrieve too much garbage/irrelevant chunks?

Comparison
Context Recall
Need: "Q3, Q4"
Got: "Q3"
Result: ❌ Missing Q4
Context Precision
Query: "Return Policy"
Got: "Policy, CEO Bio..."
Result: ❌ Garbage

Diagnostic Guide

Use RAGAS scores to pinpoint the failure mode and apply the correct fix.

Action Plan
ScoreIssueFix
Low Faithfulness LLM Hallucinating Stricter prompts, better model
Low Relevancy Off-topic answer Improve prompt (few-shot)
Low Recall Missing info Query rewriting, Hybrid search
Low Precision Garbage chunks Add Reranker, Improve chunking
> 0.80 Excellent Validate with humans

Key Takeaways

✅RAGAS Metrics: Faithfulness, Relevancy, Recall, Precision.
✅Faithfulness: Detects hallucinations (Answer vs Context).
✅Retrieval: Recall checks completeness; Precision checks relevance.
✅Use Scores: Diagnose specific bottlenecks (Prompt vs Retrieval).
