Cheat Sheet

Technique 1: Query Rewriting

Problem: User queries are ambiguous ("What's the timeline?"). Generic embeddings retrieve irrelevant chunks.

Solution: LLM rewrites the query to be specific before embedding.

"Timeline" → "Q3 Project Release Timeline"

Technique 2: HyDE

Hypothetical Document Embedding.

Query is too short to embed well. Instead, generate a fake answer first, then embed the fake answer.

The fake document is dense and informative, leading to better vector matches than the sparse query.

Technique 3: Hybrid Search

Dense + Sparse. Embeddings miss keywords; Keywords miss semantics. Use both.

  • Dense: Finds "Compensation" (Semantic).
  • Sparse (BM25): Finds exact "CEO" (Keyword).

Merge: Use Reciprocal Rank Fusion (RRF) or Weighted Scoring.

Full Advanced Pipeline

1. Query Rewrite → Clarify.

2. Multi-Query Retrieval → Dense + Sparse for each.

3. Deduplicate → Merge lists.

4. Rerank → Cross-encoder scoring.

5. Generate → LLM answer.

Decision Tree

How to choose the right technique?

  • Ambiguous? → Query Rewrite.
  • Very Short? → HyDE.
  • Need Keywords? → Hybrid.
  • Noisy Results? → Rerank.

Key Takeaways

✅Rewrite Queries: Fix ambiguity before searching.
✅HyDE: Generate fake answers to improve embedding.
✅Hybrid: Combine Dense (Meaning) + Sparse (Keywords).
✅Rerank: Validate retrieved chunks before answering.
