13. Fine-Tuning: PEFT vs LoRA vs QLoRA

Parameter Efficient Fine-Tuning • Quantization • Hardware
Cheat Sheet

The Problem: Full Fine-Tuning

Full fine-tuning updates ALL parameters (e.g., 175B). This requires massive GPU RAM (700GB) and is prohibitively expensive ($1M+).

💡 PEFT (Parameter-Efficient Fine-Tuning): Train only a tiny subset (0.001%) of parameters. 10x cheaper, 10x less memory, 90% accuracy.

LoRA (Low-Rank Adaptation)

Instead of updating weight matrix W, add a small perturbation ΔW = A × B.

  • W: Original weights (frozen, huge).
  • A, B: Small, trainable matrices (low-rank).
  • Result: Trainable params drop from 175B to ~3.7M (0.01%).

QLoRA (Quantized LoRA)

Quantization: Compress weights to lower precision (Float32 → Int4). Reduces memory from 56GB to 7GB.

Combines the small trainable LoRA matrices with a compressed base model, enabling fine-tuning on consumer GPUs (RTX 4090/3090).

Hardware Decision Tree

Choose your fine-tuning method based on your GPU VRAM and Budget.

  • $1M Budget? (100 GPUs) → Full Fine-Tune.
  • 24GB VRAM? (RTX 4090) → LoRA (Rank 16).
  • 16GB VRAM? (RTX 4070) → QLoRA.
  • < 16GB VRAM? → Cloud API.

End-to-End Pipeline

  1. Setup: Hardware (RTX 4090), Dataset (500 examples).
  2. Load: Quantize base model to Int4.
  3. Train: Add LoRA adapters (Rank 16). Train for ~2 hours.
  4. Merge: Write LoRA weights into base model.
  5. Deploy: Run custom chatbot.

Key Takeaways

✅PEFT: Don't retrain all 175B. Train 3.7M instead (0.01%).
✅LoRA: Best balance of cost/accuracy for most. Adds A × B matrices.
✅QLoRA: Compress base to Int4. Fits on RTX 4090/3080. Accessible.
✅Hardware: High Budget → Full FT. 24GB → LoRA. 16GB → QLoRA.
