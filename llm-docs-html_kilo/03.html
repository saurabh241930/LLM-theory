03. Prompt Engineering

Techniques • Reasoning • Structured Output
Cheat Sheet

Prompt Components

A good prompt is structured. It consists of a System Prompt, Context, Examples (Few-Shot), the Current Task, and Output Instructions.

💡 Structure improves consistency and control.

Context / Background

Zero-Shot vs. Few-Shot

Zero-Shot

Direct question. Fast, low token usage. Good for simple tasks.

Few-Shot

Provide examples (Input → Output). Teaches the model format and nuance. Higher accuracy.

Approach Comparison
Zero-Shot
"Classify: Good?"
→ Positive
Few-Shot
Ex 1: "Love it" → Pos
Ex 2: "Bad" → Neg
"Classify: Good?"
→ Positive

Chain-of-Thought (CoT)

Force the model to "think step by step". Reduces errors in logic and math by making intermediate reasoning explicit.

Zero-Shot CoT

Simply adding "Let's think step by step" to a zero-shot prompt boosts accuracy significantly.

Reasoning Flow
Step 1: Identify variables
Step 2: Apply formula
Step 3: Final Calculation

ReAct (Reason + Act)

Used for Agents. The model generates a Thought, then takes an Action (Tool), observes the result, and loops.

  • Thought: "I need to search the web."
  • Action: Search("Capital of France")
  • Observation: "Paris"
Agent Loop
LLM
Continuous loop until final answer

Structured Output

Constrain the model to produce specific formats (JSON, XML) to make parsing programmatic and reliable.

JSON Mode

API feature that guarantees valid JSON output, reducing hallucinations of syntax errors.

Formatting
{
  "type": "entity",
  "name": "John",
  "role": "admin"
}

Prompt Chaining

Break complex tasks into a sequence of simpler prompts. The output of Prompt 1 becomes the input for Prompt 2.

  • Benefit: Easier reasoning per step.
  • Benefit: Error recovery (restart just one step).

Common Pitfalls

  • Vague Instructions: "Summarize this" → "Summarize in 3 bullets".
  • No Examples: Hard for complex outputs like JSON without a sample.
  • Inconsistency: Mixing formats confuses the model.
Risks
⚠️ Vague
⚠️ No Examples
⚠️ Too Long
⚠️ Mixed Format

Key Takeaways

✅Cheapest Lever: Improving prompts costs nothing but yields high returns.
✅Examples Help: Few-shot > Zero-shot for consistency.
✅Reasoning: "Let's think step by step" catches logic errors.
✅Structure: Specify format (JSON) for easy parsing.
