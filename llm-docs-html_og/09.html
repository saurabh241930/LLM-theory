What is Function Calling?

Function calling allows LLMs to generate structured tool calls (JSON) instead of just text. This enables them to query databases, call APIs, or perform actions.

💡 Bridges the gap between LLM reasoning and external systems.

The Schema (JSON)

The LLM needs a Schema to know what tools exist. This JSON defines the function name, description, parameters, and required fields.

Key: Descriptions must be specific. "Get data" is bad; "Get current weather for a city" is good.

Function Definition
{
  "name": "get_weather",
  "desc": "Get weather for city",
  "params": {
    "city": { "type": "string" },
    "required": ["city"]
  }
}

The Function Calling Loop

  1. User Query: "Weather in NYC?"
  2. LLM Decision: Needs tool. Generates JSON call.
  3. System Execution: Calls API with params.
  4. Result Feed: Returns API output to LLM.
  5. Final Gen: LLM synthesizes answer.

Pitfalls & Best Practices

  • Vague Desc: "Get data" → Use specific descriptions.
  • Missing Required: API might crash without args.
  • Infinite Loops: Tool A calls Tool B which calls Tool A.

Key Takeaways

✅Tool Use: LLM generates structured JSON to trigger APIs.
✅Schema is King: Precise names/descriptions guide the LLM.
✅The Loop: Query -> Tool -> Result -> Synthesize.
✅Agents: Chain multiple function calls for complex tasks.
