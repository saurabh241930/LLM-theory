CHEAT SHEET

1. What is LangGraph?

LangGraph = Graph-based state machine for agents. Better than LangChain for complex workflows.

LangChain: Linear chains (A→B→C)

Limited state, hard to loop back

LangGraph: Graph execution with state (loops possible)

Full control, multi-step, conditional routing

LangGraph = Explicit state control + graph routing

2. Key Components: Nodes, Edges, State

Node: Function processing state. Calls LLM, tools, or decisions.

Edge: Connection between nodes. Can be conditional or direct.

State: Central data (messages, results, context). All nodes see it.

State is shared → all agents access same facts

Example state: messages[], current_task, tool_results

3. Simple Agent: The Loop

Flow: Start → LLM (decide) → Tool (execute) → Loop

Step 1: LLM reads state, decides action

Step 2: Tool executes, adds result to state

Step 3: Loop back (conditional edge) until done

Step 4: Return final answer

4. Supervisor Pattern: Route to Specialists

Problem: One agent can't specialize. Solution: Route to experts

Supervisor LLM: Reads query, decides routing

Specialist Agents: Code expert, math expert, research expert

Query → Supervisor picks best specialist → Specialist solves → Return answer

Each specialist has own tools and prompts

5. Collaborative Pattern: Sequential Agents

Flow: Agent A → Agent B → Agent C → Answer

Each agent refines previous work. No supervisor bottleneck.

Example: Researcher gathers → Analyst filters → Writer summarizes

Key: All agents share state, see full history, pass refined results

Better for sequential refinement workflows

🎯 Key Takeaways

  • LangGraph = Stateful graph execution (better than LangChain chains)
  • Nodes process state, Edges route between them
  • Supervisor pattern routes to specialists (best for domain-specific questions)
  • Collaborative pattern passes results sequentially (best for refinement)
  • State is shared across all agents (defines what agents can access)
  • Three memory layers: Short (context), Long (vector DB), Shared (state)
  • Always define clear state schema so all agents understand available data
