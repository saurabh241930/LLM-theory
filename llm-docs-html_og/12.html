12. Strategy: Prompt vs RAG vs Fine-Tune

Decision Making • Cost Analysis • Mistakes
Cheat Sheet

The Three Levers

You have three main ways to improve LLM outputs: Prompting (Instruction), RAG (Knowledge Retrieval), and Fine-Tuning (Behavior Change).

  • Prompting: Free, fast, immediate. Best for simple tasks.
  • RAG: Adds knowledge without retraining. Best for docs/FAQs.
  • Fine-Tuning: Changes style/behavior. Expensive. Last resort.

The Decision Tree

Don't jump straight to fine-tuning. Follow the logic:

  1. Does the model lack KNOWLEDGE? (e.g., specific docs)
    • YES → Use RAG.
    • NO → Continue.
  2. Does the model lack STYLE/BEHAVIOR? (e.g., tone, strict rules)
    • YES → Use Fine-Tuning.
    • NO → Continue.
  3. Can you solve with PROMPTS? (e.g., few-shot)
    • YES → Use Prompting.
    • NO → Fine-Tune (Last Resort).

The Common Mistake

The Trap: People jump straight to fine-tuning because it sounds powerful.

Reality: 80% of problems are solved by Prompting. 15% by RAG. Only 5% require Fine-Tuning.

Real-World Scenarios

  • Support Chatbot: External Knowledge (FAQs) → RAG.
  • Medical Triage: Strict Behavior (Rules) → Fine-Tune.
  • Code Gen: External Libs + Style → RAG + Fine-Tune.

Key Takeaways

✅Prompting First: Free, fast, solves ~80% of problems.
✅RAG Second: Adds knowledge without retraining. Best for docs.
✅Fine-Tune Last: Expensive. Only for specific behavior/style changes.
✅Decision Logic: Lack Knowledge → RAG. Lack Style → Fine-Tune.
