2. Infinite Loops: Detection & Fix

Problem: Agent calls same tool repeatedly, never progresses.

get_weather("NYC") → get_weather("NYC", detailed=true) → get_weather("NYC", extended=true) → loops forever

Solution 1: Set max_steps limit (5-10) ⭐ ESSENTIAL

Solution 2: Detect repeated actions

if current_action == last_action: force different or end

Solution 3: Track state changes

if state == prev_state: stop (no progress)

3. Hallucinated Tools: Force Valid Choices

Problem: LLM invents a tool that doesn't exist.

Available: [get_weather, get_news] | LLM tries: get_stock_price ← DOESN'T EXIST!

Solution 1: Include tool list in prompt

Solution 2: Use JSON mode with validation

Solution 3: Error recovery: Send error message, let LLM retry

Solution 4: Better tool descriptions → more context, better choices

Include list Tell LLM all tool names
Describe well Use cases, not just name
JSON mode Validate before execute
Error recovery Let LLM retry

4. Wrong Tool Selection: Guide Better

Problem: Valid tool, but wrong one for task.

Task: "USD to EUR rate?" | Picks: search_web (slow) | Should: get_exchange_rate

Solution 1: Few-shot examples

Show correct tool usage patterns

Solution 2: Tool categories

FINANCIAL: [exchange_rate, stock_price]

Solution 3: Detailed descriptions

Include use-case examples in description

❌ Poor "search_web"
✅ Better "search_web: Search internet"
❌ No categories Mixed tools
✅ Grouped FINANCIAL tools

5. Parameter Errors: Schema Validation

Problem: Tool called with invalid parameters.

Tool: get_weather(city: str, days: int) | Bad call: get_weather(city=123, days="abc")

Solution 1: Define clear schema in prompt

Solution 2: JSON schema validation

Force LLM to output valid JSON, validate before call

Solution 3: Error recovery

"days must be 1-14" → LLM self-corrects

Solution 4: Input sanitization

Clamp to valid ranges: min(14, max(1, days))

if not is_valid_json(output): retry = True if parameter_dict["days"] > 14: parameter_dict["days"] = 14

6. Tool Failures: Graceful Degradation

Problem: Tool actually fails (API down, network error).

get_weather("NYC") → ConnectionError → crash, LLM never sees error

Solution 1: Timeout limits

If tool takes >5s, timeout. Prevent hanging.

Solution 2: Retry with backoff

Try 3 times: wait 2s, then 4s, then 8s

Solution 3: Fallback to backup

Primary API fails → use backup API

Solution 4: Tell LLM what happened

"Tool failed: API timeout" → LLM adapts

7. Lost Context & Irrelevant Answers

Lost Context: Long conversations, LLM forgets original goal.

Q: "Best laptop for coding?" [10 tool calls] A: "Here's best camera for photography"

Solution 1: Track goal explicitly

Include in every prompt: "Goal: Find best laptop for coding"

Solution 2: Summarize frequently

Every 5 steps, summarize → reset messages to summary

Irrelevant Answer: Doesn't address query

Solution 3: Validate answer against query

Before returning: Does it address original question?

✅ Do Track goal
Summarize (5 steps)
❌ Don't Let context grow
Lose track of goal
✅ Do Validate answer
Check relevance
❌ Don't Return off-topic
Assume it's right

🎯 Key Takeaways

  • Infinite loops: Set max_steps (5-10), detect repeated actions, track changes
  • Hallucinated tools: Include tool list in prompt, validate names, provide descriptions
  • Wrong tools: Use few-shot examples, tool categories, detailed descriptions
  • Parameter errors: JSON schema validation, error messages, input sanitization
  • Tool failures: Timeout + retry (with backoff) + fallbacks + tell LLM what happened
  • Lost context: Track goal explicitly, summarize frequently, validate final answer
  • Communication is key: Always tell LLM what went wrong so it can recover
