ReAct Pattern

Reason + Act = Step-by-Step Problem Solving

CHEAT SHEET

1. What is ReAct?

ReAct = Reason + Act loop. Agents solve problems step-by-step:

  • Reason: LLM thinks about problem
  • Act: Call a tool
  • Observe: See result
  • Repeat: Until done

Task example: "What's weather in NYC tomorrow?"

Handles multi-step problems better than blind tool calling

2. Four Key Components

Thought: LLM explains what's needed

"Need weather API for NYC"

Action: Tool to call + parameters

get_weather(city="NYC", days=1)

Observation: Tool result

"High 72°F, Low 60°F, Sunny"

Final Answer: Human-friendly response

"Tomorrow sunny, 72°F high"

5. Implementation Pattern

Core loop:

while steps < MAX_STEPS: response = llm.complete(messages) if "Final Answer" in response: return response action = parse_action(response) result = execute_tool(action) messages += { "role": "user", "content": f"Observation: {result}" }

Key: Always set max_steps to prevent infinite loops

6. Best Practices

✓ Set max_steps: Prevent infinite loops (10-15 steps typical)
✓ Format observations: Clear JSON output from tools
✓ Log execution: Track reasoning for debugging
✓ Error recovery: Try different tool if one fails
✓ Clear tool descriptions: Help LLM pick right tool
✓ Max Steps
Safe limit (10)
✓ Good Tools
Clear descriptions

7. Common Pitfalls

Infinite Loops Calling same tool repeatedly
Wrong Tools Poor tool descriptions
Bad Format Observations not JSON
No Recovery Crashes on first error
❌ Pitfalls:
No step limit
Silent failures
Unclear tools

🎯 Key Takeaways

  • ReAct Loop = Thought → Action → Observation → Repeat
  • Multi-step problems need reasoning between tool calls
  • Always set max_steps to prevent infinite loops
  • Clear tool descriptions help LLM pick the right one
  • Format observations clearly (JSON) for LLM to understand
  • Log every step for debugging when things go wrong
