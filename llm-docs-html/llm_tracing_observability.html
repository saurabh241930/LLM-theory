LLM Tracing & Observability Stack

Why Datadog isn't enough, and how to instrument Agentic chains

The Blind Spot of Traditional APM

In standard software engineering, tools like Datadog or New Relic track Latency and HTTP 500 errors. If an endpoint takes 200ms and returns a 200 OK, the APM reports total system health.

This fails catastrophically for GenAI. An LLM endpoint can return a 200 OK with perfect latency, but the actual text payload might be a hallucinated disaster that tells a user to drink bleach. Furthermore, in an Agentic system, a single user request might trigger 8 different LLM API calls and 4 database queries behind the scenes. You must track the exact prompts, retrieved chunks, and token counts of every intermediate step.

The Anatomy of an LLM Trace

Modern observability tools (like LangSmith or Langfuse) capture nested hierarchical traces. Every execution is assigned a UUID (Trace ID), and every sub-step (Span) is attached to it.

[TRACE] User Query: "What is my account balance?" Total: $0.04 | 2.5s
[SPAN] Vector Search (Pinecone) 150ms
Retrieved 3 chunks: ["Account requires API key...", ...]
[SPAN] LLM Call (GPT-4o) - Router Logic 800ms | 150 tokens
System: "Determine next action." | Output: "CALL_TOOL(get_balance)"
[SPAN] API Tool Execution ERROR 403 | 120ms
"Invalid API Key for User ID 992"
[SPAN] LLM Call (Claude 3.5 Sonnet) - Final Answer 1.2s | 400 tokens
Output: "I cannot retrieve your balance because your API key is invalid."

If the user clicks "Thumbs Down" on the UI, that feedback rating is immediately piped back and bound to this exact Trace ID. A developer can later pull up all "Thumbs Down" traces and see exactly which step in the chain failed.

The Modern Observability Stack

LangSmith

Built by LangChain. The absolute industry standard for deep tracing and dataset creation.

Langfuse

Open-source alternative. Incredible UI for tracing and attaching user feedback scores.

Helicone

Acts as a proxy gateway. You just change the OpenAI base URL, and it logs everything instantly.

OpenTelemetry (OTel)

The vendor-neutral standard. Standardizing spans so they can be exported to standard Datadog.
