Advanced React & Next.js Architecture

RSC, Concurrent Features, Query Caching & Micro-Frontends

1. React Query (TanStack Query)

React Query manages asynchronous state (server state) independently of client state. It handles caching, deduplication, and background updates natively.

staleTime vs cacheTime (gcTime)

staleTime (default: 0): The duration until data is considered "stale". If data is fresh, React Query will serve it directly from the cache without refetching from the network.

cacheTime / gcTime (default: 5 mins): The duration inactive data remains in memory. Once a query has no active observers for this duration, it is garbage collected.

Optimistic Updates

Updating the UI instantly before the server mutation actually succeeds. If the server request fails, the cache is rolled back to its previous state.

2. React Server Components (RSC) vs Client Components

Server Components render exclusively on the server and transmit a serialized representation of the UI to the client, shipping zero JavaScript bundle weight.

graph TD subgraph Server SC1[Server Component - fetch DB directly] SC2[Server Component - read filesystem] end subgraph Client Boundary ("use client") CC1[Client Component - useState, onClick] CC2[Client Component - Browser APIs] end SC1 --> CC1 SC1 --> SC2 CC1 -. children props .-> SC2

Data Fetching Waterfalls

In traditional client-side fetching (e.g., useEffect inside components), nested components create a waterfall: Parent fetches -> Parent Renders -> Child Mounts -> Child fetches. RSC eliminates this because the server resolves all components and data fetches simultaneously or in parallel before streaming the result to the browser.

3. Next.js App Router Mechanics

The App Router shifts from page-based routing to folder-and-file-based routing, heavily utilizing React Suspense and RSC.

  • layouts & templates: layout.tsx preserves state during navigation (does not unmount). template.tsx remounts completely on navigation.
  • loading.tsx & error.tsx: Native Error Boundaries and Suspense fallbacks. When placed in a folder, Next.js automatically wraps the nested page.tsx in <Suspense fallback={...}> and <ErrorBoundary>.

Parallel & Intercepting Routes

Parallel Routes (@folder): Render multiple pages simultaneously in the same layout (e.g., a Dashboard with @analytics and @team rendered conditionally).

Intercepting Routes ((..)folder): Loading a route within the current layout context (e.g., clicking a photo in a feed opens a Model over the feed, but refreshing the page loads the photo exactly on its own distinct, full-page route).

4. useTransition & useDeferredValue

React 18 Concurrent Mode allows React to interrupt heavy rendering tasks to ensure user interactions remain responsive.

Interruptible Rendering

Without Concurrent features, rendering is a synchronous, blocking operation resulting in UI freezes.

useTransition: Used for state updates that trigger heavy calculations or fetches. You mark an update as a "transition" so React knows it's low priority. Example: Typing in a search bar instantly updates the input value (Urgent), but rendering the 10,000 search results is wrapped in startTransition (Low Priority).

useDeferredValue: Similar to debouncing but without a fixed timer. It receives a value and returns a deferred version of it. React will attempt to re-render with the deferred value when the thread is idle.

5. React Testing Library (RTL)

RTL enforces testing components from a user's perspective rather than testing implementation details (like component state or internal methods).

Best Practices

  • Screen Queries: Always use screen.getByRole or screen.getByText over getByTestId. If a user cannot find it visually by role/text, you have an accessibility issue.
  • userEvent vs fireEvent: userEvent perfectly simulates full browser interactions (firing focus, keydown, input, keyup sequentially), whereas fireEvent just dispatches a single raw DOM event.
  • What NOT to test: Do not test third-party library behavior, do not test internal state variables, and avoid snapshot testing as they are brittle and often ignored by devs.

6. useRef Deep Cases

useRef is essentially an escape hatch from React's declarative, state-driven paradigm. It acts as a mutable container whose identity remains stable across re-renders.

DOM Refs vs Mutable Instance Vars

While usually used to grab classical DOM elements (e.g., inputRef.current.focus()), useRef is also used to store mutable data that should not trigger a re-render when mutated (unlike useState). Example: Storing a setInterval ID to clear it later.

forwardRef & useImperativeHandle

By default, components do not expose DOM nodes to their parents. forwardRef encapsulates a component allowing the parent to pass a ref down.

useImperativeHandle restricts what the parent can do with that Ref. Instead of giving the parent the raw DOM node, you expose a controlled API, e.g., ref.current.customFocusMethod().

7. Custom Hooks Patterns

Custom hooks extract stateful logic from components allowing for clean composition.

  • useDebounce: Returns a debounced version of a rapidly changing value (often paired with `useEffect` timeouts).
  • usePrevious: Utilizes a useRef and a useEffect. Because Effects run after render, the Ref holds the value from the previous render cycle, which can be returned synchronously before the Effect updates it.
  • useEventListener: Safely adds/removes event listeners on mount/unmount entirely abstracting the messy cleanup logic out of the core component.

8. Error Boundaries & Portals

Error Boundaries

React requires Error Boundaries to be built as Class Components utilizing static getDerivedStateFromError() (to render fallback UI) and componentDidCatch() (to log errors to Sentry/Datadog).

Functional components cannot natively be error boundaries yet. The best practice is using the react-error-boundary package which provides reset fallback strategies (e.g., "Try Again" buttons).

Portals & Event Bubbling

createPortal renders a React element into a completely different DOM node (like document.body). This is universally used for Modals, Tooltips, and Dropdowns to escape CSS overflow: hidden and z-index stacking context issues.

Senior Signal: Even though a Portal takes the element physically out of the React DOM tree, React Event Bubbling still functions exactly as if the element was inline. Clicking a Modal rendered in the <body> via Portal still bubbles the `onClick` up to its original React Parent Component.

9. Micro-Frontend Architecture

Scaling huge frontend apps across multiple autonomous teams involves breaking down monolithic React apps.

graph TD Shell[Shell Application / Host] TeamA[Remote App A: Product Catalog] TeamB[Remote App B: Checkout Service] Shell --> TeamA Shell --> TeamB

Webpack Module Federation

Module Federation allows dynamic loading of JavaScript code from another distinct build at runtime. A "Shell" application lazily loads remote React bundles seamlessly (e.g., rendering a Next.js App component inside an old Create-React-App host).

It explicitly handles dependency sharing: If the Shell already loaded React 18, it tells the Remote Apps to use the host's version, preventing massive bundle duplication over the wire.
