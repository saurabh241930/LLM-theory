System Design: Enterprise AI Search (like Glean)

Multi-Tenant Semantic Search with Strict Access Control

Q85: Design an LLM-powered Enterprise Search System (like Glean)

A dual-pipeline architecture (Ingestion vs. Retrieval) featuring hybrid search (BM25 + Dense Vectors), where strict multi-tenant data isolation is enforced via metadata filtering before the LLM ever sees context.

Enterprise search is vastly more complicated than a simple PDF Q&A app. The defining challenge is Access Control Lists (ACLs). If the CEO asks about "Project Alpha," they should see private financial docs. If an intern asks, they must not see those docs, even if the semantic similarity is huge.

Data is continuously pulled from various highly secure internal sources (Slack, Confluence, Google Drive).

3. Key Design Decisions & Reasoning

A. Multi-Tenant Data Privacy (Pre-Filtering vs Post-Filtering)
  • Post-filtering is a trap: If you fetch the top 10 most relevant chunks and then check permissions, all 10 might be forbidden (e.g., restricted HR docs). Your system returns 0 results, even if the 11th chunk was a valid public summary.
  • Pre-filtering (The Solution): Modern Vector DBs support metadata filtering. You inject the user's ACLs directly into the vector search query: `similarity_search(query_vector, filter={role: {$in: user_roles}})`
B. Hybrid Search (BM25 + Dense Vectors)
  • Dense embeddings (semantic search) are great for meaning ("how do I fix a bug"), but terrible at specific keyword matching (e.g., searching for a specific Jira ticket number like "ENG-8912" or an employee's exact surname).
  • Solution: Both systems are run. BM25 catches the exact keywords, and semantic embeddings catch the intent. The lists are merged using Reciprocal Rank Fusion (RRF) before being sent to the LLM.
Depth (Senior Signal) - Near Real-Time Indexing:

Unlike web crawlers, Enterprise Search must reflect updates instantly. If a user gets fired, they must lose access to Confluence search immediately. If a new PR is merged, it should be searchable within seconds. You achieve this by having both a Batch Pipeline (nightly deep crawls to rebuild damaged indexes) and a Streaming Pipeline (Kafka/Webhook listeners for immediate `INSERT`/`DELETE` vector operations).

Trap / Watch Out: Interviewers will press you on Hallucinations pointing to wrong source documents. Ensure you include a mechanism for Provenance / Citations. The prompt must instruct the LLM: "You must answer using ONLY the provided context. After every sentence, append a citation [DocID]." The UI then maps [DocID] to a clickable hyperlink.
