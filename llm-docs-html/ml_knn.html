k-Nearest Neighbors (k-NN)

Lazy learning based on proximity
k-NN
Type
Instance-based (Lazy)
Memory

Metric
Euclidean / Manhattan

1. Summary

k-NN is a "lazy" algorithm because it doesn't learn a model. Instead, it stores the entire dataset and, at prediction time, finds the k closest training points to the new input. The output is decided by a majority vote of these neighbors.

4. Choosing the right "k"

  • Small k (e.g., 1): Overfitting. Noise will capture the prediction.
  • Large k: Underfitting. The model becomes too generic and biased towards the dominant class.

5. Scenarios & Use Cases

👍 Good For

  • Small to medium datasets.
  • Recommendation systems (similar users/items).
  • Anomaly detection.

❌ Bad For

  • Large data: Calculating distance to millions of points is slow.
  • High dimensions: "Curse of Dimensionality" makes distances less meaningful.

6. Real-World Case Study: Retail Recommender Systems

The Problem: An e-commerce site wants to suggest products based on "Users who bought this also liked..."

The Solution: k-NN finds the 'k' most similar users in the database.

  • Features: Purchase history, pages viewed, time spent on site.
  • Prediction: Recommend items purchased by the 'k' nearest neighbors that the current user hasn't seen yet.

Why k-NN? It is Instance-Based. It doesn't need to "learn" a global model of all shoppers; it simply looks for local patterns in the immediate neighborhood of a specific user, making recommendations feel highly personalized.
