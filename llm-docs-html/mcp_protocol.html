Model Context Protocol (MCP)

The Open Standard for Universal AI Integrations

1. Why MCP? The N*M Integration Problem

Before MCP, every AI Host (Claude, IDEs, Custom Apps) had to write custom integrations for every Data Source (GitHub, Slack, Databases). This created a fragmented ecosystem.

graph LR subgraph Traditional H1[Host A] --> I1[Custom Git] H1 --> I2[Custom Slack] H2[Host B] --> I3[Custom Git] end subgraph MCP_Standard HA[Host A] -- MCP --> S1(MCP Server: Git) HB[Host B] -- MCP --> S1 HC[Host C] -- MCP --> S2(MCP Server: SQL) end

MCP solves this by providing a universal interface. Write a server once, and any MCP-compliant host can use it.

2. Core Architecture

MCP consists of three main components:

  • Host: The AI application (e.g., Claude Desktop, Cursor, local agent) that initiates the connection.
  • Client: The part of the host that maintains the protocol connection to a server.
  • Server: A lightweight process that exposes data and tools to the host.

Q: How do Host and Server communicate?

They use JSON-RPC 2.0. Communication can happen over two main transports:

  • Stdio: The host spawns the server as a child process (best for local tools).
  • HTTP/SSE: Communication over the network (best for remote services).

3. The Three Primitives: Resources, Prompts, Tools

Primitive Analogy Definition
Resources Reading a file Read-only data the model can inspect (Files, DB schemas, API logs).
Prompts Templates Re-usable instruction sets that guide the model (e.g., "Analyze this code").
Tools Executing code Functions the model can call to perform actions (Write file, Execute query).

4. Building an MCP Server

You can build MCP servers using the official TypeScript or Python SDKs.

// TypeScript MCP Server Example
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";

const server = new Server({ name: "my-tool", version: "1.0.0" }, { capabilities: { tools: {} } });

server.setRequestHandler(ListToolsRequestSchema, async () => ({
  tools: [{ name: "get_weather", description: "Get weather info", inputSchema: {...} }]
}));

const transport = new StdioServerTransport();
await server.connect(transport);

5. MCP Senior Interview Questions

Q: How does MCP improve RAG architectures?

Instead of manual ETL pipelines where data is pre-chunked into a vector DB, MCP allows Just-In-Time retrieval. The model can browse a live file system or query a live database via MCP Tools/Resources, ensuring it uses the most current context without syncing issues.

Q: What are the security implications of MCP?

Since MCP servers can execute local commands or access sensitive files, sandboxing is critical. Most hosts allow users to approve specific "MCP Tools" before execution, and servers should ideally follow the principle of least privilege.
