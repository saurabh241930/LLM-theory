1. Merge Intervals

Intuition: This pattern is used to deal with overlapping intervals. The key is to Sort the intervals based on their start time. Once sorted, we only need to compare the end of the current interval with the start of the next.

Time Complexity: O(n log n) Space Complexity: O(n)
Algorithm Visualizer: Interval Merging
Click Next to begin merging overlapping intervals.
intervals.sort(key=lambda x: x[0])
merged = [intervals[0]]
for current in intervals[1:]:
last = merged[-1]
if current[0] <= last[1]:
last[1] = max(last[1], current[1])
else:
merged.append(current)

2. Cyclic Sort

Intuition: Used when the input numbers are in a range from 1 to n. Instead of a full sort, we iterate and swap each number to its correct index (i.e., num at index num-1).

Time Complexity: O(n) Space Complexity: O(1)
Algorithm Visualizer: Cyclic Index Positioning
Sort numbers in-place in O(n) by placing them at their correct indices.
while i < len(nums):
j = nums[i] - 1
if nums[i] != nums[j]:
nums[i], nums[j] = nums[j], nums[i]
else:
i += 1

3. Two Heaps (Median of a Stream)

Intuition: Finding the median of a growing stream of numbers is tricky. We use two heaps: a Max-Heap for the smaller half and a Min-Heap for the larger half. The median is either the top of the larger heap or the average of both tops.

Time Complexity: O(log n) per add Space Complexity: O(n)

heappush(max_heap, -heappop(min_heap))

4. Subsets (Backtracking / BFS)

Intuition: To generate all subsets, we start with an empty set. For each number, we clone all existing subsets and add the new number to them. This creates a branching factor of 2 at each step.

Time Complexity: O(2^n) Space Complexity: O(2^n)
Algorithm Visualizer: Power Set Generation
Watch the power set grow exponentially as we process [1, 2, 3].
subsets = [[]]
for num in nums:
for i in range(len(subsets)):
sub = list(subsets[i])
sub.append(num)
subsets.append(sub)
