Metadata Filtering in Vector DBs

Pre-filter, Post-filter, and modifying HNSW for Hybrid Queries

The Challenge of "Where" and "When"

Vector search is incredible at finding concepts ("How do taxes work?"), but terrible at strict constraints ("Show me documents from 2023 where Author=Alice"). A vector database doesn't understand boolean logic. To fix this, we attach JSON Payloads / Metadata to the vectors, and use filtering.

However, running a metadata filter alongside a vector search creates a massive algorithmic headache. Do you filter first, or search first?

Post-Filtering (The Naive Way)

1. Run the vector search first (e.g., fetch the Top 100 closest vectors).
2. Discard the ones that don't match the metadata (e.g., not from 2023).

Fatal Flaw: If only 1% of your database is from 2023, fetching the Top 100 vectors might yield exactly 0 vectors from 2023. You completely destroy your Recall. The database returns an empty array, even though the data exists.

Pre-Filtering (The Slow Way)

1. Scan the entire database for metadata matches (e.g., find all 1 million documents from 2023).
2. Run the vector search only on that subset.

Fatal Flaw: You completely lose the speed of the HNSW (Hierarchical Navigable Small World) graph. You are forced to do a "brute-force" KNN calculate across 1 million documents, destroying latency.

The Soltuion: Intelligent HNSW Payload Filtering

Modern Vector Databases (like Qdrant, Milvus, and Pinecone) solve this by modifying the HNSW index structure itself at build time.

Layer 3 (Top): Broad navigation
Layer 2: Finer navigation
Layer 1: Payload-Aware Connections
Layer 0 (Base): Actual Vectors

When the database builds the navigational graph, it ensures that nodes with different metadata categories maintain separate navigational pathways. During inference, it uses Single-Stage Filtering: as the algorithm hops from node to node through the graph looking for your answer, it simply refuses to step onto any node that violates your metadata filter.

This retains the 10-millisecond speed of HNSW while guaranteeing 100% accuracy on the boolean constraints, completely solving the Recall problem associated with post-filtering.
