Distributed Training Strategies

DP, TP, PP, and FSDP: Training models larger than a single GPU can hold

The Mathematics of GPU Memory

A flagship Nvidia H100 GPU has 80GB of VRAM. A relatively small 70B parameter model in 16-bit precision requires 140GB of VRAM just to store the weights. When you add the optimizer states (Adam requires 2x the memory of the weights) and the massive gradient matrices during the forward/backward passes, training a 70B model requires over 1,200 GB of VRAM.

Because you physically cannot fit the training state on one GPU, you must distribute the math across a cluster of GPUs (Nodes).

The 4 Pillars of Distributed Parallelism

Data Parallelism (DP)
GPU0: Full Model
GPU1: Full Model

The simplest approach. Replicate the identical model onto every GPU. Split the massive training dataset into chunks. GPU0 trains on batch A, GPU1 trains on batch B. At the end of the step, all GPUs average their gradients together over the network (All-Reduce) and update their weights.

Fatal Flaw: If the model doesn't fit onto a single GPU, DP is entirely useless.

Pipeline Parallelism (PP)
GPU0: Layers 1-40
GPU1: Layers 41-80

Split the model vertically. Put the first 40 Transformer blocks on GPU0, and the last 40 on GPU1. GPU0 calculates the forward pass, sends the activations over the network to GPU1, and then sits totally idle waiting for GPU1 to calculate the Loss and send the gradients backwards.

Fatal Flaw: Massive idle time (the "Pipeline Bubble"). Solved via Micro-batching, but incredibly difficult to optimize correctly.

Tensor Parallelism (TP)
GPU0: Left Half of Matrix
GPU1: Right Half

Split the model horizontally. Literally take a massive 4096x4096 weight matrix inside a Linear Layer and slice it in half. Both GPUs calculate their half of the mathematical dot product simultaneously, and instantly synchronize the results.

Fatal Flaw: Requires insane, ultra-low latency network synchronization (NVLink) because the GPUs must communicate during every single mathematical operation. Cannot be done across different server nodes.

The Modern Standard: FSDP / ZeRO 👑

Fully Sharded Data Parallelism (PyTorch FSDP) or DeepSpeed ZeRO-3.

Instead of doing complex math splitting like TP or PP, FSDP is just a massive memory hack applied to basic Data Parallelism. It mathematically slices the massive Optimizer States, the Gradients, and the Weights across all the GPUs. GPU0 holds 10% of the model, GPU1 holds 10%, etc.

During the forward pass, when GPU0 needs the weights for Layer 1, it rapidly downloads those specific weights from the other GPUs, computes the math, and instantly deletes the weights from its memory to save space. It treats the collective VRAM of all GPUs in the cluster as one giant shared hard drive. This is how massive models like LLaMA 3 400B are trained today.
