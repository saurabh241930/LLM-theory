Data Splitting & Cross Validation

Proper Model Evaluation & Preventing Data Leakage

The Golden Rule of ML

Never evaluate a model on the data it was trained on.

If you test a student on the exact homework questions they already saw, you don't know if they learned the concepts or just memorized the answers. The same applies to ML models.

The Standard Split (Train / Val / Test)

Training Set (70-80%)
Val Set (10-15%)
Test Set (10-15%)
  • Training Set: Used to compute gradients and update weights.
  • Validation Set: Used during training to tune hyperparameters (like learning rate) and trigger Early Stopping.
  • Test Set: Locked away until the absolute final step. Used exactly once to estimate real-world performance.

⚠️ Data Leakage (The Silent Killer)

Data leakage occurs when information from outside the training dataset is used to create the model. If your model gets 99% accuracy but fails in production, you probably have leakage.

  • Target Leakage: Including a feature that is a proxy for the label (e.g., predicting if a user gets a loan, and including their "monthly loan payment amount" as a feature).
  • Temporal Leakage: Using future data to predict the past in time-series forecasting.
  • Leaky Preprocessing: Fitting a StandardScaler or TF-IDF Vectorizer on the entire dataset before splitting. Always split first, then fit on train, transform on val/test.

k-Fold Cross Validation

A single Train/Val split can be lucky or unlucky. k-Fold splitting divides the dataset into k equal chunks (folds). The model is trained k separate times, using a different fold as the validation set each time.

Example: 5-Fold Cross Validation

Fold 1 (Val)
Fold 2 (Train)
Fold 3 (Train)
Fold 4 (Train)
Fold 5 (Train)
Fold 1 (Train)
Fold 2 (Val)
Fold 3 (Train)
Fold 4 (Train)
Fold 5 (Train)
... repeats 5 times

Result: Every single data point gets to be in the validation set exactly once. The final score is the average of the k validation scores, providing a much more robust estimate of true performance.

Stratified k-Fold

If your dataset is highly imbalanced (e.g., 99% benign, 1% fraud), a random split might create a fold with zero fraud cases. Stratified splitting guarantees that every fold has the exact same ratio of classes as the overall dataset.
