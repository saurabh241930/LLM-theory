2. The Python Event Loop in FastAPI

Q: How does the Event Loop handle 10k concurrent requests?

By never blocking. When a request hits an `await` (e.g., DB query), the loop suspends that function and picks up the next one immediately.
Senior Signal: If you use a blocking library (e.g., legacy `requests`) inside an `async def` function, you freeze the entire event loop for all users. Performance drops to 1 QPS per process.

3. Advanced Dependency Injection

Q: Why is FastAPI's DI system better than manual instantiation?

It handles the lifecycle and caching of dependencies (like DB sessions or Auth users) automatically for each request.
# Advanced DI with sub-dependencies
async def get_db_session(): ...

async def get_current_user(db: Annotated[Session, Depends(get_db_session)]): ...

@app.get("/")
async def read_root(user: Annotated[User, Depends(get_current_user)]): ...

4. Senior FastAPI & Python Interview Masterclass (50+ Questions)

This section covers the most frequently asked questions in high-level FastAPI and Python interviews, ranging from event loop internals to production scaling patterns.

Hot

Q1: Explain generators and when you'd use them in a backend context.

Generators are data streams that use yield to produce values lazily. They maintain their internal state without loading everything into memory (O(1) space).

Backend usage: Handling massive database cursors, streaming large file uploads, or processing paginated API results without hitting VRAM or RAM limits.

def stream_logs():
    with open("large_log.txt") as f:
        for line in f:
            yield line # Processed one by one
Hot

Q2: What's GIL and how does it affect your FastAPI service under load?

The Global Interpreter Lock (GIL) ensures only one thread executes Python bytecode at a time. While it's fine for I/O-bound tasks (where Threads wait for DBs), it severely hurts CPU-bound tasks (encryption, ML inference).

Effect under load: Multiple workers (Gunicorn) bypass this by using separate processes, but within a single worker, multi-threading won't speed up CPU calculation.

Hot

Q3: How does FastAPI handle async vs sync route handlers? When would you use each?

FastAPI handles async def and regular def differently to maximize throughput:

  • async def: Runs directly on the Event Loop. Use this for non-blocking I/O (DB queries, API calls using httpx).
  • def: Runs in an external Threadpool. Use this for blocking I/O (libraries like requests or bs4) or CPU-bound tasks, so they don't freeze the Event Loop.
@app.get("/async")
async def async_route():
    await asyncio.sleep(1) # Non-blocking
    return {"message": "Async"}

@app.get("/sync")
def sync_route():
    time.sleep(1) # Blocking, but run in threadpool
    return {"message": "Sync"}

Q4: What happens if you block the event loop in an `async def` route?

The entire application stops responding to ALL users until the blocking task is finished.

Example: Using time.sleep(10) instead of await asyncio.sleep(10) inside an async def will freeze the server.

Q5: Explain Depends() — how does dependency injection work in FastAPI?

FastAPI's DI system uses Depends() to declare requirements for a route. It handles the lifecycle and caching of shared resources like database sessions or security context.

async def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

@app.get("/items")
async def read_items(db: Session = Depends(get_db)):
    return db.query(Item).all()

Q6: What's the difference between Pydantic v1 and v2, and how does it affect FastAPI?

Pydantic v2 was a complete rewrite in Rust, focusing on performance and strictness. For FastAPI users:

  • Performance: Serialization and validation are up to 20x faster.
  • Breaking Changes: .dict() is now .model_dump(), and schema() is model_json_schema().
  • Native Serialization: V2 handles JSON serialization natively, removing the need for jsonable_encoder in many cases.

Q7: Can dependencies be cached in FastAPI?

Yes, by default, FastAPI caches the result of a dependency for the duration of a single request. If multiple routes/sub-dependencies use the same Depends(get_db), the function only runs once.

Q8: Gunicorn vs Uvicorn — What is the production standard?

Run Gunicorn as a process manager with Uvicorn workers (uvicorn.workers.UvicornWorker). Gunicorn provides process management (restarts, signals), while Uvicorn handles the ASGI protocol.

Q9: How do you differentiate between a Path and Query parameter?

Path parameters are part of the URL (/items/{id}). Query parameters are appended after the ? (/items?start=0&limit=10). In FastAPI, you declare them as function arguments.

Q10: How do you implement JWT auth with OAuth2PasswordBearer in FastAPI?

Use OAuth2PasswordBearer to define the token URL. FastAPI will automatically check the Authorization: Bearer header. You then manually decode and validate the JWT using libraries like pyjwt or python-jose.

from fastapi.security import OAuth2PasswordBearer
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")

@app.get("/users/me")
async def read_users_me(token: str = Depends(oauth2_scheme)):
    user = decode_jwt(token)
    return user

While usually an LLM term, in FastAPI datasets, this refers to using cursors/offsets to prevent loading huge tables into RAM at once.

Q12.1: Difference between @staticmethod, @classmethod, and instance method.

  • Instance Method: Takes self; accesses instance data.
  • @classmethod: Takes cls; used for Factory patterns or modifying class state.
  • @staticmethod: No extra args; just a function scoped to a class for organization.
Hot

Q12.2: How does Python's async/await work under the hood? What is an event loop?

Python uses Generators and Coroutines. When await is hit, it pauses the coroutine and yields control back to the Event Loop (an infinite loop that checks for ready sockets/I/O).

Senior Signal: The event loop uses select() or epoll() (on Linux) to efficiently wait for multiple file descriptors simultaneously.
Core

Q13: Middleware vs Dependencies — when do you use which?

  • Middleware: Runs on every request/response. Best for logging, CORS, or global headers. Cannot easily access route-specific dependencies.
  • Dependencies: More granular. Can be applied to specific routes, routers, or globally. Have access to the route's parameters and DI graph.

Q14: Explain `Annotated` in FastAPI 0.95+.

Allows combining types with metadata: db: Annotated[Session, Depends(get_db)]. It is now the recommended standard.

Hot

Q15: How do you handle background tasks without blocking the response?

FastAPI provides a BackgroundTasks class. You add a function and its arguments to the task list; FastAPI will return the response to the client first and then execute the function.

from fastapi import BackgroundTasks

def write_log(message: str):
    with open("log.txt", "a") as f:
        f.write(message)

@app.post("/send-notification/{email}")
async def send_notification(email: str, background_tasks: BackgroundTasks):
    background_tasks.add_task(write_log, f"notification sent to {email}")
    return {"message": "Notification sent in the background"}

Q16: How to handle Global Exception Handlers?

Use @app.exception_handler(MyException). It allows catching custom business exceptions and returning clean JSON.

Q17: What is Pydantic `BaseModel` vs `TypeAdapter`?

BaseModel is for objects (classes). TypeAdapter (V2) is for validating generic types like List[str] or Dict without a class wrapper.

Q18: Explain `TestClient` vs `httpx.AsyncClient`.

TestClient is sync (based on requests). Use httpx.AsyncClient with pytest-asyncio for testing async routes effectively.

Q19: How to protect against SQL Injection in FastAPI?

Use Pydantic for validation and an ORM (SQLAlchemy/SQLModel) that uses parameterized queries. Never use f-strings to build SQL.

Q20.1: What are Python dataclasses vs Pydantic models? When to use which?

  • Dataclasses: Built-in, light, minimal validation. Best for internal data structures inside the domain.
  • Pydantic: Validation-first, parses JSON/Envs. Best for Boundary Objects (API inputs/outputs).

Q20.2: Explain context managers and how yield works in them.

Used for resource management (DBs, files). With @contextmanager, code before yield is the setup, and code after yield is the teardown (cleanup). 

@contextmanager
def transaction():
    print("BEGIN")
    try:
        yield
    finally:
        print("COMMIT/ROLLBACK")

Q21: How do you handle database connection pooling in a FastAPI app?

Use an async engine with a built-in pooler (like SQLAlchemy's AsyncEngine or tortoise-orm). Initialize the pool in the lifespan event to ensure it's created once and shared across all workers. 

from sqlalchemy.ext.asyncio import create_async_engine
engine = create_async_engine(DATABASE_URL, pool_size=20, max_overflow=0)

Q22: How to handle CORS in FastAPI?

Use CORSMiddleware. You must specify allow_origins, allow_methods, and allow_headers to allow frontend apps to call your API.

Q23: What are Pydantic Generic Models?

Models that use typing.Generic. Useful for standardizing API responses: ResponseModel[T] where data is of type T.

Q24: Explain `UploadFile` vs `bytes` for file uploads.

bytes loads the whole file into RAM (bad for large files). UploadFile uses a spooled file (stored on disk if it exceeds a memory limit), which is much safer.

Q25: How to use `Form` data?

Import Form from fastapi. Unlike JSON, form data is sent as application/x-www-form-urlencoded. You need python-multipart installed.

Q26: What is `HTTPBasic`?

A security scheme for Basic Auth (Username/Password in headers). FastAPI provides a helper to parse and validate it automatically.

Q27: How to implement Rate Limiting?

FastAPI doesn't have it built-in. Use slowapi or a custom middleware with **Redis** to track request counts per IP/API key.

Q28: Explain `StreamingResponse`.

Used to stream content (like video or large CSVs) to the client without loading it all into memory first.

Q29: What is `Field` in Pydantic?

Used to add extra validation or metadata to model attributes, like ge=0 (greater than zero) or description="The user's age".

Q30: How to use `ConfigDict` in V2?

In Pydantic V2, the internal Config class is replaced by model_config = ConfigDict(...). It controls behavior like extra='forbid' or from_attributes=True.

Q32: How to mount Static Files?

Use StaticFiles from fastapi.staticfiles and app.mount("/static", ...). Useful for serving CSS/JS if you aren't using a CDN.

Q33: What is a Sub-dependency?

A dependency that itself has dependencies. FastAPI resolves the entire tree automatically: User -> Current_User -> DB_Session.

Q34: How to use `Request` object directly?

Inject request: Request into your route. Useful for accessing raw headers, client IP, or the app state directly.

Q35: Explain `WebSockets` in FastAPI.

FastAPI supports two-way real-time communication. Use @app.websocket("/ws") and await websocket.accept().

Q36: What is `RedirectResponse`?

Returns a 301/302/307 status code to redirect the user to another URL, either internal or external.

Q37: How to handle `RequestValidationError`?

Override the default handler to return custom JSON instead of the standard 422 Unprocessable Entity error.

Q38: Explain `Security` vs `Depends`.

Security is like Depends but it also takes Scopes. Useful for fine-grained OAuth2 permissions (e.g., "read", "write").

Q39: What is `OAuth2PasswordRequestForm`?

A helper that parses username and password from a standard form-post, which is required by the OAuth2 specification.

Q40: How to use `Cookie` and `Header` params?

Import them from fastapi. They work like Query or Path, extracting data from specific parts of the request.

Q41: Explain `Starlette`'s `TestClient` internals.

It uses requests to make calls against the ASGI app without actually running a network server, making tests very fast.

Q42: How to handle `Multi-part` forms?

When sending both files and JSON, you must use Form fields. You cannot send a raw JSON body alongside multipart/form-data easily without a wrapper.

Q43: What is `Recursive Models`?

Pydantic models that reference themselves (e.g., a Comment with a list of replies: List['Comment']). Use model_rebuild() in V2.

Q44: How to use `Computed Field` in Pydantic V2?

Use the @computed_field decorator on a property. It will be included in the serialized output (JSON).

Q46: How do you implement rate limiting in FastAPI without a third-party service?

Use an in-memory Fixed Window or Token Bucket algorithm using a dict (for single worker) or Redis (shared across workers) inside a FastAPI dependency.

from fastapi import HTTPException, Request
import time

_rates = {}
def rate_limit(request: Request):
    ip = request.client.host
    now = time.time()
    if ip in _rates and now - _rates[ip] < 1:
        raise HTTPException(status_code=429)
    _rates[ip] = now

Q47: How do you implement streaming LLM responses via Server-Sent Events in FastAPI?

Use EventSourceResponse from sse_starlette or a custom generator with StreamingResponse and text/event-stream header.

async def streamer():

        yield chunk
return StreamingResponse(streamer(), media_type="text/event-stream")

Q48: Explain the Circuit Breaker pattern — how would you implement it in a FastAPI GenAI service?

Prevents a failing service (like an OpenAI API timeout) from cascading. If error rates exceed a threshold, the "circuit opens" and immediately returns a fallback/error for a cooldown period.

Implementation: Use the pybreaker library or a custom decorator around your LLM call dependency.

Scale

Q49: How do you handle multi-tenant isolation in a FastAPI + PostgreSQL app?

  • Shared Schema: Every table has a tenant_id column (RLS in Postgres is recommended).
  • Dynamic Schema: Every tenant has its own Postgres Schema.
  • Dynamic DB: Every tenant has a separate database.
FastAPI Implementation: Inject tenant_id via a dependency that extracts it from the JWT or Header.

Q50: How would you design a job queue for async LLM processing at scale?

Use Celery or ARQ with Redis as a broker. The FastAPI route submits the task and returns a task_id. A separate set of GPU-enabled workers consumes the queue.

@app.post("/v1/process")
async def process():

    return {"task_id": task.id}

Q51: How do you stream LLM tokens using FastAPI's StreamingResponse?

Yield tokens from your generator and return a StreamingResponse with the text/event-stream media type for Server-Sent Events (SSE). 

@app.post("/generate")
async def generate():
    async def token_generator():
        for token in ["Hello", " from", " FastAPI!"]:
            yield token
    return StreamingResponse(token_generator(), media_type="text/plain")
Hot

Q52: What are the best practices for handling long-running inference in FastAPI?

Offload the inference to a Worker Process (Celery/Ray) and use a Task ID pattern. The client polls the status, or the server sends a webhook upon completion. Never run heavy models directly on the FastAPI event loop thread.
