2. The Python Event Loop in FastAPI

Q: How does the Event Loop handle 10k concurrent requests?

By never blocking. When a request hits an `await` (e.g., DB query), the loop suspends that function and picks up the next one immediately.
Senior Signal: If you use a blocking library (e.g., legacy `requests`) inside an `async def` function, you freeze the entire event loop for all users. Performance drops to 1 QPS per process.

3. Advanced Dependency Injection

Q: Why is FastAPI's DI system better than manual instantiation?

It handles the lifecycle and caching of dependencies (like DB sessions or Auth users) automatically for each request.
# Advanced DI with sub-dependencies
async def get_db_session(): ...

async def get_current_user(db: Annotated[Session, Depends(get_db_session)]): ...

@app.get("/")
async def read_root(user: Annotated[User, Depends(get_current_user)]): ...

4. Senior FastAPI Interview Q&A

Q: What is the difference between `async def` and `def` routes in FastAPI?

FastAPI runs async def functions directly on the event loop. It runs regular def functions in an external ThreadPool. Use def for legacy blocking code and async def for modern non-blocking libraries.

Q: How do you handle middleware performance bottlenecks?

Middleware runs on every request. Keep logic light (auth, headers). For heavy lifting, use Dependencies or Background Tasks.
