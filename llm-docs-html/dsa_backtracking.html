Backtracking: The Decision Tree Explorer

State Space Trees, Recursive Pruning, and the "Choose-Act-Undo" Loop

1. Technical Deep-Dive: DFS with Memory

Backtracking is a systematic way to iterate through all configurations of a search space. It's built on a "Brute Force" search but optimized with Pruning.

The 3 Pillars: Choice, Constraint, Goal

Every backtracking algorithm follows this blueprint:

  1. Choice: What decision do I make next? (e.g., Which number to put in a Sudoku cell?)
  2. Constraint: Is the choice valid? (e.g., Is the number already in that row?)
  3. Goal: Have I reached a solution? (e.g., Is the board full?)

The "Undo" Step: Because we pass state by reference (like a list in Python or Object in JS), you MUST remove your choice before returning from the recursion so the next branch starts with a clean slate.

A. Subsets (Power Set)

Finding all possible combinations of elements from a set.

JavaScript
Python
function subsets(nums) {
    let result = [];
    
    function backtrack(start, currentPath) {
        // Goal: Every path is a valid subset
        result.push([...currentPath]);
        
        for (let i = start; i < nums.length; i++) {
            // 1. Choose
            currentPath.push(nums[i]);
            // 2. Recurse
            backtrack(i + 1, currentPath);
            // 3. Undo (Backtrack)
            currentPath.pop();
        }
    }
    
    backtrack(0, []);
    return result;
}
def subsets(nums: List[int]) -> List[List[int]]:
    res = []
    
    def backtrack(start, path):
        # Goal: Capture the state at every node
        res.append(list(path))
        
        for i in range(start, len(nums)):
            # 1. Choose element
            path.append(nums[i])
            # 2. Recurse to next index
            backtrack(i + 1, path)
            # 3. Undo (POP) to explore others
            path.pop()
            
    backtrack(0, [])
    return res

B. N-Queens (Constraint Logic)

The classic visualization of pruning. If a queen is placed illegally, we don't even explore that entire subtree.

function solveNQueens(n) {
    let board = Array.from({length: n}, () => '.'.repeat(n).split(''));
    let res = [];

    function backtrack(row) {
        if (row === n) {
            res.push(board.map(r => r.join('')));
            return;
        }
        for (let col = 0; col < n; col++) {
            if (isValid(row, col)) {
                board[row][col] = 'Q';
                backtrack(row + 1);
                board[row][col] = '.'; // Backtrack
            }
        }
    }
    // isValid implementation omitted for brevity
    backtrack(0);
    return res;
}
def solveNQueens(n):
    res = []
    cols = set()
    posDiag = set() # (r + c)
    negDiag = set() # (r - c)
    board = [["."] * n for _ in range(n)]

    def backtrack(r):
        if r == n:
            res.append(["".join(row) for row in board])
            return

        for c in range(n):
            if c in cols or (r+c) in posDiag or (r-c) in negDiag:
                continue # Pruning: Skip invalid placements

            cols.add(c); posDiag.add(r+c); negDiag.add(r-c)
            board[r][c] = "Q"
            backtrack(r + 1)
            # Backtrack state
            cols.remove(c); posDiag.remove(r+c); negDiag.remove(r-c)
            board[r][c] = "."

    backtrack(0)
    return res
