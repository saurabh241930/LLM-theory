JS & TS Mastery

Deep Internals, Modern JS & TypeScript Generics

1. JavaScript Core & Internals

Q1: Explain 'Closures' in JavaScript.

A closure is the combination of a function bundled together (enclosed) with references to its surrounding state (the lexical environment). This allows a function to access variables from an outer scope even after that outer scope has closed.

Q2: What is the 'Event Loop' in the browser?

The event loop is a mechanism that handles asynchronous callbacks. It constantly checks if the call stack is empty; if so, it pushes the first task from the task queue to the stack.

Q3: Explain Prototype Inheritance.

Every object in JS has a prototype property. When we try to access a property, if not found on the object, JS looks it up in its prototype, following the 'prototype chain' until it finds it or hits null.

Q4: Difference between `null` and `undefined`?

`undefined` means a variable has been declared but not assigned a value. `null` is an assignment value representing 'no value' or 'nothing'.

Q5: What is 'this' keyword in JS?

The value of `this` depends on how a function is called. In global scope, it's `window` (or `global`). In an object method, it's the object. In an arrow function, it's lexically inherited.

Q6: Explain `call`, `apply`, and `bind`.

`call` and `apply` invoke a function immediately with a specific `this`. `bind` returns a new function with `this` permanently set.

Q7: What is 'Hoisting'?

JS moves variable and function declarations to the top of their scope before code execution. Only declarations are hoisted, not initializations.

Q8: `let` vs `const` vs `var`?

`var` is function-scoped and hoisted. `let` and `const` are block-scoped and not accessible before declaration (Temporal Dead Zone). `const` cannot be reassigned.

Q9: What are 'Promises'?

A Promise represents the eventual completion (or failure) of an asynchronous operation and its resulting value (Pending, Fulfilled, Rejected).

Q10: `async` vs `await`?

`async` makes a function return a promise. `await` pauses execution until a promise is settled, making async code look synchronous.

Q11: What is 'Currying'?

A technique of transforming a function that takes multiple arguments into a sequence of functions that each take a single argument.

Q12: Explain 'Debouncing' and 'Throttling'.

Debouncing delays execution until a certain time has passed since the last call. Throttling limits execution to once every X milliseconds.

Q13: What is 'Rest' vs 'Spread' operator?

Rest (`...`) collects multiple elements into an array. Spread (`...`) expands an array/object into individual elements.

Q14: Shallow vs Deep Copy?

Shallow copy only copies the first level of references. Deep copy recursively copies all levels (e.g., using `JSON.parse(JSON.stringify(obj))` or `structuredClone`).

Q15: What are 'WeakMap' and 'WeakSet'?

They are collections where keys (Map) or values (Set) must be objects and are held 'weakly'—they don't prevent garbage collection if no other references exist.

Q16: Explain 'Event Delegation'.

A technique where you attach a single event listener to a parent element to handle events for all its children, using 'bubbling'.

Q17: What is the 'Execution Context'?

An environment in which JS code is executed. It contains the variable object, scope chain, and `this` value.

Q18: Explain 'Map' vs 'Object'.

Map keys can be any type; Object keys are strings/symbols. Maps maintain insertion order and have better performance for frequent additions/removals.

Q19: What is 'Strict Mode'?

`'use strict'` enables a constrained version of JS that catches common coding bloopers and prevents certain 'unsafe' actions (like assigning to undeclared variables).

Q20: What are 'Symbols' in JS?

A unique and immutable data type used for object property keys to avoid name collisions.

Q21: Explain 'Generator Functions'.

Functions that can be paused and resumed (`function*` with `yield`). They return an iterator.

Q22: What is 'The Temporal Dead Zone'?

The period between a variable's block start and its declaration (for `let`/`const`) where accessing it throws a ReferenceError.

Q23: How does 'Garbage Collection' work in JS?

Primarily 'Mark and Sweep'. It starts from 'roots' and marks reachable objects; anything unmaked is swept away.

Q24: What is 'Macrotasks' vs 'Microtasks'?

Microtasks (Promises, queueMicrotask) run after the current task and before the next Macrotask (setTimeout, I/O).

Q25: Explain 'Proxy' and 'Reflect'.

Proxy lets you intercept and redefine operations for an object (like GET/SET). Reflect provides methods for the interceptable JS operations.

Q26: What are the benefits of TypeScript over JS?

Static typing, early error detection, IDE support (intellisense), and better documentation through types.

Q27: Difference between `interface` and `type`?

Both define shapes. Interfaces support declaration merging and are generally preferred for public APIs. Types support unions, intersections, and aliases.

Q28: What is 'Any' vs 'Unknown'?

`any` turns off type checking. `unknown` is safe; you must perform type checking (narrowing) before using the value.

Q29: Explain 'Generics'.

Allow creating reusable components that work with a variety of types while maintaining type safety (e.g., `function identity(arg: T): T`).

Q30: What is 'Type Guarding' / 'Type Narrowing'?

Using code to refine a broader type into a more specific one (e.g., `typeof`, `instanceof`, or custom `is` functions).

Q31: Explain 'Partial', 'Pick', and 'Omit' utility types.

`Partial` makes all keys optional. `Pick` creates a type with selected keys K. `Omit` removes keys K.

Q32: What is 'ReadOnly' vs 'const' in TS?

`const` is for variable reassignment. `ReadOnly` makes properties of an object immutable at the type level.

Q33: Explain 'Union' and 'Intersection' types.

Union (`A | B`) means a value can be either A OR B. Intersection (`A & B`) means it has ALL properties of both.

Q34: What is 'keyof' operator?

Produces a union of an object's keys (e.g., `keyof {id: 1, name: 'a'}` is `'id' | 'name'`).

Q35: Explain 'Mapped Types'.

Creating new types by iterating over keys of an existing type (e.g., `{ [K in keyof T]: boolean }`).

Q36: What is 'Conditional Types'?

Types that depend on a condition (`T extends U ? X : Y`).

Q37: Explain 'as const' (const assertions).

Marks a value as immutable and its type as literal (not inferred broadly).

Q38: What are 'Decorators'?

A special kind of declaration used to modify classes, methods, or properties at design time (experimental feature).

Q39: Difference between 'Abstract Class' and 'Interface'?

Abstract classes can contain implementation; interfaces cannot. Classes can implement multiple interfaces but only extend one abstract class.

Q40: What is 'Record'?

A utility type for creating a map-like type where keys are K and values are T.

Q41: Explain 'NonNullable'.

Removes `null` and `undefined` from a union type.

Q42: What is 'Tuple' in TS?

An array with a fixed number of elements where each element has a specific type (e.g., `[string, number]`).

Q43: Explain 'Lookup Types' (Indexed Access Types).

Accessing a property type of another type using `T[K]` (e.g., `User['id']`).

Q44: What is 'Template Literal Types'?

Types that use template string syntax to create new string union types.

Q45: Difference between `enum` and `const enum`?

`enum` exists at runtime. `const enum` is inlined at compile time, saving bundle size.

Q46: What is 'Type Assertions' (`as` keyword)?

Telling the compiler 'Trust me, I know the type better than you do'. It doesn't perform runtime casting.

Q47: Explain 'Infer' keyword.

Used within conditional types to infer a type variable (e.g., `T extends (...args: any[]) => infer R ? R : any`).

Q48: What is 'Ambient Modules' (`declare`)?

Used to describe the shape of libraries not written in TS, usually in `.d.ts` files.

Q49: Explain 'Module Resolution' strategies.

How TS finds files for imports: 'Node' (following node_modules) or 'Classic' (legacy).

Q50: Why use 'Utility Types'?

To transform existing types into new variations without duplication, keeping the codebase DRY and maintainable.
