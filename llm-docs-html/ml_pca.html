PCA (Principal Component Analysis)

Complexity reduction via dimensionality projection
PCA
Type
Unsupervised (Feature Eng)
Goal
Maximize Variance Retention
Technique
Eigen-Decomposition

1. Summary

PCA is used to reduce the number of features in a dataset while keeping as much information (variance) as possible. It transforms original, correlated features into a new set of uncorrelated features called Principal Components.

2. Visual Flow

4. Intuition

Imagine a cloud of data points. PCA finds the direction (axis) along which the data is most spread out. This becomes "Principal Component 1." The second axis must be perpendicular to the first and capture the next most variance.

5. Scenarios & Use Cases

🌟 Benefits

  • Data Compression: Reduces redundant features.
  • Visualization: Project high-dimensional data to 2D.
  • Denoising: Removes subtle noise.

🚩 Drawbacks

  • Requires scaling (Standardization).
  • Assumes linear relationships.
  • Loss of literal meaning for features.

6. Real-World Case Study: Image Compression & Facial Recognition

The Problem: A surveillance system needs to store 1 million faces but doesn't have the terabytes of storage required for raw high-res images.

The Solution: PCA (Eigenfaces) reduces each face image from 10,000 pixels down to just 50 principal components.

  • Input: 100x100 pixel grid (10k features).
  • Output: 50 numerical values that represent the "core" features of that face.

Why PCA? Variance Retention. PCA identifies that most pixels in a face photo are redundant (e.g., background or skin texture). By keeping only the components with high variance, we can reconstruct the face with 95% accuracy using only 0.5% of the original data!
