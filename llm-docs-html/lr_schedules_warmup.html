Learning Rate Schedules & Warmup

Why Transformers crash on step 1, and the necessity of Cosine Annealing

The Mathematics of the Warmup Phase

If you take a Convolutional Neural Network (CNN) like ResNet, you can usually start training immediately with your maximum Learning Rate (LR). If you try this with a Transformer, the loss will instantly hit NaN (Not a Number) and the model will permanently break on Step 1. Transformers mandate a Warmup Phase.

Why do Transformers need Warmup?

At initialization, a Transformer is incredibly fragile. The Layer Normalization statistics (mean and variance) are completely untrained and highly erratic. Furthermore, the Softmax function inside the Attention layers is violently volatile when fed large random initial weights.

If a large gradient (created by a high learning rate) hits this unstable system on step 1, the LayerNorm/Softmax outputs explode, multiplying into infinity via the Chain Rule. Warmup solves this by starting the Learning Rate at exactly 0.0 and linearly increasing it to the maximum value over the first 1,000 to 10,000 steps, allowing the normalization statistics to safely settle.

Cosine Annealing Decay

Once the model survives the warmup and reaches the maximum Learning Rate, we must gradually lower the LR. If we don't, the model will bounce recklessly around the optimal minimum of the loss landscape, never actually settling into it.

The undisputed industry standard for LLM pretraining is Cosine Annealing. After warmup, the learning rate drops following the curve of a cosine function, until it reaches about 10% of the maximum value right as the training run finishes.
