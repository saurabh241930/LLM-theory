Rotary Positional Embedding (RoPE)

Interactive visualization of how LLMs inherently understand time and sequence

The "Bag of Words" Problem

A pure Transformer Attention mechanism has absolutely no concept of Time or Sequence. If you feed it "The dog bit the man" and "The man bit the dog", the math inside the QKV matrices calculates the exact same semantic overlap because the sets of words are identical. The original 2017 paper solved this by injecting Sinusoidal math directly into the token embeddings.

Modern models (LLaMA 3, Mistral) use Rotary Positional Embedding (RoPE). Instead of adding a fixed integer to a variable, they physically rotate the semantic vector on a 2D plane by an angle corresponding to its position index.

RoPE Rotation Simulator

Vector Angle Θ = 0°

Notice how Token 0 points directly Right. As the token moves further into the prompt sequence (Position 1, 2, 3...), the actual semantic embedding vector is mathematically forced to rotate m * θ degrees.

The brilliance of RoPE is that the relative distance between Token 3 and Token 8 is identical to the relative distance between Token 13 and Token 18. The dot product preserves this perfect relative rotation!
