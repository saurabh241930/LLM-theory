What is an Embedding?

Embeddings are dense numerical representations of meaning. We convert raw text into arrays of numbers where each dimension captures a semantic aspect.

💡 Similar meanings result in vectors that are geometrically close in the high-dimensional vector space.

Why Embeddings Matter

Traditional (Keyword)

Matches exact strings. "Cat" misses "Feline". High precision, low recall.

Embedding (Semantic)

Matches based on vector proximity. "Feline" is mathematically close to "Cat". "Pizza" is far.

Cosine Similarity

Measures the angle between vectors, not distance. Robust to document length (magnitude).

cos(θ) = (A · B) / (||A|| × ||B||)
  • 1.0: Identical direction
  • 0.0: Unrelated
  • -1.0: Opposite
Vector Angles
Smaller Angle = Higher Similarity

Vector DB Indexing

Naive search (O(n)) is too slow. Vector DBs use ANN (Approximate Nearest Neighbor) indexing:

  • HNSW: Graph-based layers.
  • IVF: Partitions space into clusters.

Result: O(log n) via spatial partitioning.

Common Pitfalls

  • Wrong Model: General text fails for code/legal.
  • Truncation: Chunks too long for model context.
  • Drift: Old vectors in DB don't match new model.
  • Precision: High cosine ≠ semantic truth. Use reranking.

Key Takeaways

✅Semantics as Numbers: Embeddings map meaning to math.
✅Cosine Similarity: Measures angle, not magnitude.
✅Indexing: HNSW/IVF enables speed (O(log n)).
✅Quality: Depends on chunking and model choice.
