1. Architecture: Control Plane vs Workload

Kubernetes is a distributed system designed for Self-Healing and Autoscaling.

The Control Plane (Brains)

  • etcd: Consistent and highly-available key-value store for cluster data.
  • kube-apiserver: The gateway to the cluster.
  • kube-scheduler: Decides which node a pod should run on.
  • kube-controller-manager: Runs controller processes (Node, Deployment, CronJob).

3. Services & Ingress (L4 vs L7)

  • ClusterIP: Internal-only communication between pods.
  • NodePort: Exposes the service on a static port on each Node's IP.
  • LoadBalancer: Exposes the service externally using a cloud provider's load balancer.
  • Ingress: Manages external access to services, typically HTTP/HTTPS (L7). Provides routing, SSL termination, and name-based virtual hosting.
