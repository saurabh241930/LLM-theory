1. Summary

Naive Bayes is based on Bayes' Theorem. It is called "Naive" because it assumes that all features are independent of each other. It calculates the probability of each class and picks the one with the highest likelihood.

3. Mathematics

P(C|x) ∝ P(x|C) * P(C)

In Gaussian Naive Bayes, we assume features follow a Normal Distribution:

P(x|C) = (1 / √(2πσ²)) * exp(-(x-μ)² / 2σ²)

5. Real-World Case Study: Email Spam Filtering

The Problem: Gmail needs to decide if an incoming mail is "Spam" or "Inbox" in milliseconds.

The Solution: Multinomial Naive Bayes calculates word probabilities.

  • Features: Frequency of words like "Gift", "Lottery", "Meeting", "Invoice".
  • Calculation: P(Spam | "Lottery") is high; P(Inbox | "Meeting") is high.

Why Naive Bayes? Speed and Scalability. Even with a vocabulary of 50,000 words, Naive Bayes only needs to perform simple additions and multiplications, making it fast enough to process billions of emails daily across the globe.
