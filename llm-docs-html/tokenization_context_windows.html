In the architecture of Large Language Models (LLMs), tokenization is the process of translating human-readable text into a sequence of discrete units called tokens. These tokens are more granular than words but more meaningful than individual characters. For a model, text like "Hello, world!" is decomposed into numeric IDs (e.g., [15496, 11, 28357, 28808]) which are then processed as tensors.

How Tokenization Works

Modern LLMs use sub-word tokenization algorithms such as Byte-Pair Encoding (BPE). BPE iteratively merges the most frequent pairs of characters or character sequences in a corpus to build a vocabulary. This allows the model to handle "apple" as a single token while breaking "tokenization" into sub-pieces like ["token", "iz", "ation"].

The Context Window

The context window is the maximum number of tokens an LLM can process in a single inference step. It defines the model's "total memory," encompassing the system prompt, user messages, and historical context. As the conversation progresses, the combined token count must remain below this fixed ceiling.

The Quadratic Complexity Trap

Historically, increasing context windows was difficult due to Attention mechanisms scaling at $O(n²)$—meaning doubling the context quadrupled the compute cost. Modern optimizations like Flash Attention and Ring Attention have mitigated this, enabling the "long-context" era where entire codebases can be analyzed at once.

What Happens at the Limit?

When an input exceeds the model's limit, the system typically responds in one of three ways:

  • Hard Error: The API rejects the request completely.
  • Truncation: The system keeps the earliest tokens (the head) and discards the rest.
  • Sliding Window: The system keeps the most recent tokens (the tail) and drops older history, maintaining a "rolling memory."

Token Counting and Optimization

Accurately counting tokens before an API call is essential for cost management and preventing errors. A reliable "rule of thumb" is that 1,000 tokens represent approximately 750 words.

import tiktoken
enc = tiktoken.encoding_for_model("gpt-4")
text = "Hello, world!"
tokens = enc.encode(text)
print(len(tokens)) # Output: 4

Optimization Strategies

  • Summarization: Compressing 8,000 tokens of history into a 500-token summary.
  • Chunking (RAG): Splitting long docs into small, searchable pieces to only include the relevant 5% in the window.
  • Prompt Compression: Removing filler words to retain semantic meaning while reducing token count.
💡 Special Tokens: LLMs use invisible markers like <|im_start|> and <|im_end|> to separate dialogue turns. These consume tokens and must be accounted for in your total budget.

Common Pitfalls

  • Missing Token Buffer: Failing to account for the model's output tokens (the response also counts toward the window limit).
  • Inconsistent Tokenizers: Counting tokens with a GPT-3 tokenizer and sending them to a GPT-4 model.
  • Context Bloat: Including irrelevant metadata (URLs, timestamps) that wastes valuable window space.

Case Study: RSA Sales QA Dashboard

During the auditing of Roadside Assistance (RSA) sales calls, the choice of tokenizer is critical. Insurance policies contain unique alpha-numeric codes like RSA-24-PREM-DISC. A sub-word tokenizer like tiktoken ensures these aren't broken into meaningless fragments (e.g., ["R", "SA", "-", "24"]), allowing the model to recognize them as distinct insurance products. Furthermore, because call transcripts are long (30+ minutes), the 128k context window of models like GPT-4o allows the dashboard to "read" the entire conversation at once without losing the beginning of the sale when the mandatory disclosures are usually made.

# RSA Case Study: Managing Long Transcripts import tiktoken tokenizer = tiktoken.encoding_for_model("gpt-4o") transcript = "Advisor: Welcome to RSA Sales... [30 minutes of dialogue] ...End of Call." token_count = len(tokenizer.encode(transcript)) if token_count > 120000: print("Transcript exceeds standard context. Applying summarization window.") else: print(f"Audit safe: {token_count} tokens used.")

Key Takeaways

✅ Tokens are sub-word units, not whole words.
✅ Context windows define the model's operational memory depth.
✅ Quadratic scaling (Attention) makes long context computationally expensive.
✅ Use tiktoken or specific libraries to count tokens accurately before execution.
