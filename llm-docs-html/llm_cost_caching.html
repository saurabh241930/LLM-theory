Cost Optimization & Caching

Prompt caching, semantic caching, routing, and surviving the OpenAI bill

The Mathematics of LLM Bankruptcy

If your application has 10,000 Daily Active Users, and each user asks 5 questions, and every question injects a 10,000-token PDF as context using GPT-4o ($5.00 per 1M tokens), you are burning $2,500 every single day in raw API costs.

Production engineering is entirely about optimizing these variables. There are four primary levers to pull.

1. Semantic Caching

Standard caching fails because no two humans ask the identical string "What is my password?". To solve this, we use a Vector Database (like RedisVL or GPTCache) strictly as a cache.

Mechanic: Embed the user's query into a vector. Do a vector search against previously answered queries. If the cosine similarity is > 95%, immediately return the old generated answer.

Cost: $0.00 per repeated concept.

2. Prompt Caching (Provider Native)

Anthropic (Claude) and OpenAI now natively support caching massive system prompts. Instead of re-processing a 50-page PDF of instructions every single API call, the provider keeps the Key-Value (KV) matrices in memory on their own servers for 5 minutes.

Mechanic: Send the exact same prefix tokens in subsequent requests.

Cost: Reduces input token pricing by up to 50-80%.

3. Intelligent Model Routing

You do not need an IQ 150 model (GPT-4o / Claude 3.5 Sonnet) to answer "Hi, what time do you open?".

Mechanic: Use an ultra-fast, ultra-cheap classifier model (like LLaMA 3 8B or GPT-4o-mini) to evaluate the query's complexity. If it's simple, answer it immediately. If it requires deep reasoning or math, route the prompt to the expensive flagship model.

Cost: Offloading 80% of traffic to mini-models cuts total bills by 10x-20x.

4. Context Compression

Vector databases often return chunks full of HTML tags, filler words ("therefore", "however"), and redundant sentences.

Mechanic: Pass the chunks through a fast local model algorithm like LLMLingua. It deletes all non-essential tokens, turning "The quick brown fox jumps over the lazy dog" into "quick brown fox jumps lazy dog". The LLM still perfectly understands the semantic meaning, but reads 40% fewer tokens.

Cost: Slashes token volume by 30-50%.
