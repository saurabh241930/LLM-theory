1. Dijkstra's Algorithm (Weighted Shortest Path)

Intuition

Simple BFS finds the shortest path in unweighted graphs by exploring "rings" of distance. However, when edges have weights (costs), a simple hop-count isn't enough.

The Greedy Strategy: Dijkstra's always picks the "closest" node that hasn't been finalized yet.

  • Use a Priority Queue (PQ) to store nodes sorted by their current best-known distance from the start.
  • Initialize dist[start] = 0 and all others to Infinity.
  • Repeatedly pop the node u with the smallest distance. For each neighbor v, check if passing through u offers a shorter path: dist[u] + weight(u,v) < dist[v].
  • If it does, "relax" the edge, update dist[v], and push v back into the PQ.
Time Complexity: O((V+E) log V) Space Complexity: O(V + E)
