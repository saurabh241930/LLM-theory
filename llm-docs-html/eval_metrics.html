Evaluation Metrics in Depth

Precision, Recall, F1, and going beyond simple accuracy

The Problem with Accuracy

If you build a model to detect a rare disease that affects 1 in 10,000 people, a model that simply outputs "Healthy" 100% of the time will have 99.99% accuracy. But it's completely useless. Accuracy fails on imbalanced datasets.

Core Metrics

Precision

"Of all the items the model flagged as positive, how many actually were?"

Precision = TP / (TP + FP)

Optimize for this when: False Positives are expensive. (e.g., YouTube Recommendations, Spam Filtering). You don't want to show bad videos to users.

Recall (Sensitivity)

"Of all the actual positives in the real world, how many did the model find?"

Recall = TP / (TP + FN)

Optimize for this when: False Negatives are expensive. (e.g., Cancer detection, Fraud detection). You'd rather have a false alarm than let a dying patient go untreated.

F1 Score

The harmonic mean of Precision and Recall. It punishes extreme values. If Precision is 1.0 and Recall is 0.0, the regular average is 0.5, but the F1 score is 0.

F1 = 2 * (Precision * Recall) / (Precision + Recall)

Best overall metric for imbalanced classification tasks.

AUC-ROC

The Area Under the Receiver Operating Characteristic Curve. It tells you the probability that the model will rank a randomly chosen positive instance higher than a randomly chosen negative one.

  • 1.0 = Perfect model
  • 0.5 = Random guessing (useless)

Crucially, AUC-ROC is threshold-invariant. It measures the fundamental separability of the classes, regardless of what probability threshold (e.g., > 0.5) you pick for classification.

NLP & Text Generation Metrics

Traditional ML metrics don't work for generative AI. How do you measure if a paragraph is "good"?

Metric What it measures Drawbacks
BLEU n-gram overlap against a reference translation. (Precision-focused) Punishes valid synonyms. Very rigid.
ROUGE Recall-oriented n-gram overlap. (Used heavily in Summarization) Still syntax-bound. Cannot evaluate creative reasoning.
LLM-as-a-Judge Using GPT-4 to score accuracy, relevance, and tone. (RAGAS framework) Expensive, slow, and subject to LLM bias (e.g., preferring longer answers).
