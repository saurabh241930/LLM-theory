Overfitting & Underfitting

The Bias-Variance Tradeoff and Regularization Techniques

Part 1: The Core Intuition

Every ML model faces a fundamental tension called the bias-variance tradeoff:

  • Bias = error from wrong assumptions. A linear model fitting a curved dataset has high bias. It underfits.
  • Variance = sensitivity to small fluctuations in training data. A high-degree polynomial that passes through every point has high variance. It overfits.
  • Irreducible error = noise in the data itself. No model eliminates this.
Error = Bias² + Variance + Irreducible Noise

Part 3: Regularization Techniques

When you detect overfitting, these are the tools to fix it.

L2 Regularization (Ridge)

Adds λ · Σwᵢ² to the loss. Penalizes large weights by pulling them toward zero proportionally. Weights never reach exactly zero — all features stay in play, just constrained. Standard in logistic regression and as weight decay in deep learning optimizers (weight_decay in PyTorch Adam).

L1 Regularization (Lasso)

Adds λ · Σ|wᵢ| to the loss. The absolute value penalty creates a sharp corner at zero, which causes some weights to become exactly zero. This is automatic feature selection — L1 tells you which inputs don't matter. Use when you suspect many features are noise.

💡 The Interview Distinction: L2 shrinks all weights proportionally. L1 kills irrelevant features entirely. In high-dimensional embedding spaces (e.g., RAG with many metadata features), L1 is valuable for sparsity.

Dropout

During each training forward pass, randomly zero p fraction of neurons (typical p = 0.2–0.5). The network can't rely on any single neuron, so it's forced to learn distributed, redundant representations.

At inference time, all neurons are active but weights are scaled by (1−p) to maintain expected output magnitude — this is called inverted dropout. PyTorch's nn.Dropout handles this automatically. Note: standard Transformers (GPT, BERT) use dropout sparingly, mainly in attention weights and after feed-forward layers.

Early Stopping

Monitor validation loss during training. If it doesn't improve for N consecutive epochs (the patience), stop and restore the checkpoint from the best epoch.

Completely free — no math, no hyperparameters beyond patience. Always combine with model checkpointing. This is the most used regularization technique in LLM fine-tuning because it requires zero changes to the model architecture.

k-Fold Cross-Validation

Rather than using a single train/val split (which can be lucky or unlucky), k-fold splits the data into k equal parts. You train k separate times, each time holding out a different fold for validation. The final score is the average over all k validation scores.

The key insight: k-fold gives you a much more reliable estimate of how well your model will generalize. A single validation split might be 5% better or worse than reality just from random chance. k-fold averages that out.

💡 For LLMs: k-fold is expensive since training runs take hours/days. Instead, use a fixed dev set with multiple random seeds and report mean ± std. For fine-tuning evaluation, use stratified splits to ensure label balance across splits.
