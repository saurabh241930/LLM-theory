HNSW, IVF-Flat, and the Math of Approximate Nearest Neighbors (ANN)

1. HNSW: The Gold Standard Index

Hierarchical Navigable Small Worlds (HNSW) is the engine behind Pinecone, Milvus, and pgvector. It treats the vector space as a graph where nodes represent embeddings.

Why "Hierarchical"?

Instead of searching every node, HNSW creates "shortcuts" in upper layers. We start at Layer 1 (coarse), find the closest node, then drop down to Layer 0 (dense) to refine the search. This turns O(N) search into O(log N).

2. IVF-Flat: Inverted File Index

If HNSW is about connectivity, IVF-Flat is about neighborhoods. It uses K-Means to cluster vectors into 'Voronoi cells'.

  • Indexing: Chops the space into nlist clusters.
  • Search: Only searches the nprobe clusters closest to the query.
  • Tradeoff: Uses significantly less memory than HNSW but is slightly slower and less accurate.

3. The Geometry of Distance

Different models require different distance metrics. Using the wrong one causes "Semantic Drift."

Cosine Similarity: $\cos(\theta) = \frac{A \cdot B}{\|A\|\|B\|}$ — Measures angular distance. Industry standard for text.

Euclidean (L2): $d = \sqrt{\sum(a_i - b_i)^2}$ — Measures actual spatial gap. Best for image data.

Inner Product (IP): $d = A \cdot B$ — Extremely fast but sensitive to vector magnitude.

4. Index Tuning Parameters

  • M: Max number of edges per node in HNSW. Higher M = Higher accuracy, Higher Memory.
  • efConstruction: Search depth during index build.
  • efSearch: Search depth during query. High efSearch = Lower Recall errors but Higher Latency.
