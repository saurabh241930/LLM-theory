Dynamic Programming: The Art of Storing Work

Overlapping Subproblems, Tabulation, and Time-Space Trade-offs

1. Technical Deep-Dive: Why DP Exists

Dynamic Programming is simply Recursion + Memoization. It is used when an expensive calculation repeats itself multiple times in a decision tree (Overlapping Subproblems).

The Two Philosophies

1. Top-Down (Memoization): You start with the big problem and break it down. If f(n) has already been solved, you return the cached result. Benefit: Only computes required states.

2. Bottom-Up (Tabulation): You solve the smallest subproblems first (0, 1, 2...) and fill a table until you reach n. Benefit: Avoids recursion depth overhead; easier to space-optimize (e.g., only keeping last 2 variables).

How to Spot a DP Problem?

  • Keywords: "Total number of ways...", "Minimum cost to...", "Longest path...".
  • Constraint: The answer for N depends on N-1 and/or N-2.

3. Master Patterns & Dual-Language Code

A. Knapsack Pattern (0/1 DP)

Deciding "to include or not include" an item to maximize value within weight limit.

JavaScript
Python
function knapsack(weights, values, W) {
    const n = weights.length;
    let dp = Array.from({length: n + 1}, () => new Array(W + 1).fill(0));

    for (let i = 1; i <= n; i++) {
        for (let j = 1; j <= W; j++) {
            if (weights[i-1] <= j) {
                // Max(Take it, Leave it)
                dp[i][j] = Math.max(values[i-1] + dp[i-1][j - weights[i-1]], dp[i-1][j]);
            } else {
                dp[i][j] = dp[i-1][j];
            }
        }
    }
    return dp[n][W];
}
def knapsack(weights, values, W):
    n = len(weights)
    # Space Optimized 1D DP
    dp = [0] * (W + 1)
    
    for i in range(n):
        # Traverse backwards to avoid using same item twice
        for j in range(W, weights[i] - 1, -1):
            dp[j] = max(dp[j], values[i] + dp[j - weights[i]])
            
    return dp[W]

B. Longest Common Subsequence (LCS)

Finding the longest sequence of characters that appear in both strings in the same relative order.

JavaScript
Python
function longestCommonSubsequence(text1, text2) {
    const m = text1.length, n = text2.length;
    let dp = Array.from({length: m + 1}, () => new Array(n + 1).fill(0));

    for (let i = 1; i <= m; i++) {
        for (let j = 1; j <= n; j++) {
            if (text1[i-1] === text2[j-1]) {
                dp[i][j] = 1 + dp[i-1][j-1];
            } else {
                dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]);
            }
        }
    }
    return dp[m][n];
}
def lcs(s1, s2):
    m, n = len(s1), len(s2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                dp[i][j] = 1 + dp[i-1][j-1]
            else:
                # Max of ignoring char from s1 or s2
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
                
    return dp[m][n]
