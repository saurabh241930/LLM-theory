Service Mesh vs API Gateway

North-South vs East-West Traffic Management

1. API Gateway (Nginx, Kong, AWS API Gateway)

An API Gateway is explicitly an Edge component. It sits exclusively between your external clients (Web Browsers, Mobile Apps) and your internal microservices (North-South Traffic).

Primary Responsibilities

  • Edge Router: Single entry point blocking direct access to internal IPs. Handles path routing (/api/users -> User Service).
  • Auth Termination: Validates JWTs globally before traffic even hits your internal systems.
  • Rate Limiting: Denies basic DDoS vectors and limits customer API tiers globally.
  • SSL Termination: Handles HTTPS handshakes, forwarding plain HTTP internally to save CPU cycles on microservice pods.

2. Service Mesh (Istio, Linkerd)

A Service Mesh is explicitly an internal infrastructure layer. Once a request bypasses the API Gateway and enters the internal cluster, the Service Mesh manages how the internal microservices securely talk to each other (East-West Traffic).

The Sidecar Proxy Pattern

A Service Mesh achieves its goal by attaching a microscopic 'Sidecar Proxy' (e.g., Envoy) directly alongside every single application container.

Instead of the Node.js User Service calling the Python Order Service directly over HTTP, the User Service Node.js app calls its own localhost Envoy proxy. The Envoy proxy validates tracing, secures the network with mTLS, checks circuit breakers, and routes the packet to the Order Service's Envoy proxy.

3. Service Mesh Killer Features

  • mTLS (Mutual TLS): Automatically encrypts all traffic between internal services without developers writing a single line of TLS code. Perfect for zero-trust networks matching HIPAA/PCI compliance.
  • Observability & Tracing: Automatically intercepts total network latency between every pod, tracking distributed Trace IDs across the cluster seamlessly, feeding into Jaeger or Datadog natively.
  • Traffic Splitting (Canary): The proxy network can flawlessly divert exactly 5% of internal traffic to the new v2 of a microservice transparently to test for crashes before a 100% rollout.
  • Retries & Circuit Breaking: Handles complex exponential backoffs natively inside the infrastructure layer without polluting the application codebase with retry libraries.
