Query Expansion (In-Depth)

PRF, Synonyms, and LLM-Expansion vs HyDE

Beyond the Raw Prompt

Users are notoriously bad at writing search queries. They submit shorthand ("aws s3 error 403") when the vector database contains beautiful documentation ("Amazon Simple Storage Service Access Denied Exception"). Query Expansion fixes the user's laziness before hitting the database.

1. Keyword / Synonym Expansion

How it works: Uses a static dictionary or simple NLP model (like Word2Vec) to append related terms.

"car insurance" → "car auto vehicle insurance coverage"

Pros: Lightning fast, zero LLM cost.
Cons: Lacks context. Might expand "apple" to "fruit" when the user meant "iPhone".

2. Pseudo-Relevance Feedback (PRF)

How it works: The gold standard of traditional search (BM25). You run the raw query, take the top 3 results, extract the most common/important keywords from those documents, append them to the query, and search again.

Pros: Grounded entirely in your actual dataset.
Cons: Requires two full search passes, doubling latency.

3. LLM-Based Expansion

How it works: You prompt a fast LLM (like GPT-3.5 or Haiku) to explicitly brainstorm variations before searching.

"Write 3 different ways a user might ask about [original query]."

Pros: Unbeatable semantic understanding. Captures edge cases perfectly.
Cons: Adds ~500ms of LLM generation delay to every search.

Expansion vs HyDE / Step-Back

It is critical to distinguish LLM Expansion from other advanced rewriting techniques:

Query Expansion

The LLM generates more questions or keywords. (e.g., "How do I reset my router?", "WiFi restarting steps", "Router reboot guide"). You vector search all three and fuse the results. Used to cover vocabulary mismatch.

HyDE (Hypothetical Document Embeddings)

The LLM generates a fake answer to the question, and you embed the answer, not the question. (e.g., "To reset your router, unplug it for 30 seconds..."). Used when the question looks completely different in vector space than the target document.

Step-Back Prompting

The LLM generates a broader, more general question. (e.g., "Why did the 1998 NASA Mars orbiter crash?" → "What are the standard units of measurement for NASA orbital missions?"). Used when specific details trap the search algorithm, requiring physical principles to solve.
