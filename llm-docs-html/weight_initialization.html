Weight Initialization

Why starting your parameter randomizations correctly decides if a model learns or dies

The Symmetry Breaking Problem

If you initialize every weight in a neural network to `0.0` (or any identical constant number), the network will completely fail to train. Every neuron in a hidden layer will perform the exact same computation, receive the exact same gradient during backpropagation, and update to the exact same new weight. No matter how many neurons you add, the model will behave entirely as if it only had a single neuron. Weights must be initialized randomly to break symmetry.

However, if you initialize weights with random numbers that are too large, the network instantly saturates the activation functions (vanishing gradients). If you initialize them too small, the signal dies out before reaching the final layer (vanishing activations). To train deep networks, the variance of the inputs must exactly match the variance of the outputs across a layer.

Modern Initialization Strategies

Xavier (Glorot) Initialization

Designed for historically common activations like Tanh and Sigmoid. It scales random numbers dynamically based on how many neurons feed into the current layer (fan-in) and how many it feeds into (fan-out).

Var(W) = 2 / (fan_in + fan_out)

By using this exact variance for thousands of initial weights, the forward pass signal maintains its amplitude all the way through a deep network.

He Initialization

When the industry switched to ReLU activations, Xavier initialization broke. ReLU kills exactly half the network output (all negative numbers become zero), halving the variance. He initialization doubles the variance component to account for ReLU's destructive nature.

Var(W) = 2 / fan_in

LLM Weight Initialization (Transformers)

In massive architectures like GPT-4 or Llama 3, poor initialization doesn't just slow down training; it causes massive loss spikes that permanently corrupt the billions of parameters. To stabilize Transformers:

  • Modern frameworks initialize the heavily stacked residual layers specifically such that the residual branch starts at exactly `0`. This means an initialized 100-layer Transformer initially behaves exactly like a 1-layer network, making it incredibly stable.
  • Instead of relying purely on perfect initial weights to fix exploding/vanishing signals, we rely heavily on Layer Normalization (LayerNorm) or RMSNorm to forcefully scale the variance back to 1.0 after every single layer.
