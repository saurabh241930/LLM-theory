1. Temperature (T)

Temperature controls the "randomness" or "creativity" of the output by scaling the logits before the Softmax function is applied.

P(xᵢ) = exp(zᵢ / T) / Σⱼ exp(zⱼ / T)
  • Low Temperature (T → 0): Makes the distribution "sharper." The highest-scoring token becomes much more likely. Used for factual/code tasks.
  • High Temperature (T → 1+): Flattens the distribution. Makes lower-probability tokens more likely, increasing creativity/variety.
  • T = 1: Standard Softmax.

2. Sampling Strategies

Once we have a probability distribution, we need to pick a token. Pure randomness often leads to gibberish, so we use filters.

Top-K Sampling

The model sorts tokens by probability and keeps only the top K most likely tokens. It then redistributes the probability among them and picks one.

Use Case: Prevents the model from picking extremely low-probability "junk" tokens.

Top-P (Nucleus) Sampling

Instead of a fixed count (K), the model picks a cumulative probability threshold P (e.g., 0.9). It keeps the smallest set of tokens whose probabilities sum up to P.

Dynamic: If only one token is very likely, only one is kept. If many are equally likely, it keeps many. This is generally preferred over Top-K.

💡 Interview Insight (Deloitte GenAI): "What makes Top-P different?" Answer: Adaptability. Unlike Top-K which always picks K tokens regardless of the probability mass, Top-P adapts its selection size based on the model's confidence. In cases of low confidence (flat distribution), the 'nucleus' expands; in cases of high confidence (sharp distribution), it shrinks.

3. Search Algorithms

Decoding can be done one-by-one (Greedy) or by looking ahead (Beam Search).

Algorithm Mechanism Pros/Cons
Greedy Decoding Always picks the token with the absolute highest probability. Fast, but can get stuck in repetitive loops.
Beam Search Keeps N candidates (beams) at each step and picks the sequence with the highest total probability. Better quality for Translation/summarization, but computationally expensive.

4. Repetition & Frequency Penalties

These techniques penalize tokens that have already appeared in the text to prevent the model from repeating itself ("The quick brown fox, the quick brown fox...").

Senior Interview Deep-Dive

Q: Why would you set Temperature to 0 in a production RAG system?

Setting T=0 (Greedy decoding) ensures Determinism. For a RAG system answering from technical docs, you want the same factual answer every time, not a creative variation. It also makes debugging and unit testing easier.
