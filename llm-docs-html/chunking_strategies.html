1. Technical Deep-Dive: The "Context-vs-Precision" Conflict

Chunking is the most underrated lever in RAG tuning. If your chunks are too small, the model loses the "Big Picture". If they are too large, you fill the context window with "Noise" and dilute the relevant answer.

The Hierarchy of Splitting

  1. Fixed-Size: Constant token count. Fast, but dangerous (splits logical thoughts).
  2. Recursive Character: Matches paragraphs → sentences → words. Best general purpose.
  3. Semantic: Uses embeddings to find natural "topic shifts". Breaks when the similarity between sentence N and N+1 drops below a percentile.
  4. Late Chunking: Embedding the entire document first, then chunking the resulting token-level embeddings. This preserves global context within the local chunk vector.

A. Semantic Splitting (Python & JS)

Using cosine distance to find breakpoint thresholds.

Python (LlamaIndex)
JavaScript (LangChain.js)
from llama_index.core.node_parser import SemanticSplitterNodeParser
from llama_index.embeddings.openai import OpenAIEmbedding

# 1. Define embedding model for similarity check
embed_model = OpenAIEmbedding()

# 2. Configure parser: high percentile = more chunks
parser = SemanticSplitterNodeParser(
    buffer_size=1, breakpoint_percentile_threshold=95, embed_model=embed_model
)

# 3. Process document
nodes = parser.get_nodes_from_documents([doc])
import { RecursiveCharacterTextSplitter } from "langchain/text_splitter";

// Standard Recursive implementation
const splitter = new RecursiveCharacterTextSplitter({
  chunkSize: 500,
  chunkOverlap: 50,
  separators: ["\n\n", "\n", ".", "!", "?", " ", ""],
});

const docs = await splitter.createDocuments([text]);
console.log(`Document split into ${docs.length} semantic fragments.`);
The Benefit of "Parent-Child" Indexing

By splitting a document into tiny Child Chunks (100 tokens) for vector search, but returning the Parent Context (1000 tokens) to the LLM, you get the best of both worlds: High retrieval accuracy and rich reasoning context.
