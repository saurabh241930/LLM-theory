Multi-Head Attention vs. MQA and the VRAM explosion

The Mathematics of VRAM Bankruptcy

During Transformer inference, we do not want to recalculate the Key and Value matrices of past tokens for every newly generated token. Instead, we temporarily save the past tokens in GPU VRAM (the Key-Value Cache). This allows O(1) compute generation scaling, but at the cost of catastrophic memory accumulation.

Memory Status: Optimal

Multi-Head Attention (MHA): The original 2017 paper. Every single attention Query head gets its own dedicated Key head and Value head. Requires storing 64 K matrices and 64 V matrices per token.

Grouped Query Attention (GQA): Used by LLaMA-2/3. Groups 8 Query heads to share a single Key and Value head. Cuts VRAM usage by 8x with virtually no loss in intelligence.

Multi-Query Attention (MQA): Forces all 64 Query heads to share exactly 1 Key head and 1 Value head. Cuts VRAM by 64x, but can cause a minor degradation in reasoning quality.
