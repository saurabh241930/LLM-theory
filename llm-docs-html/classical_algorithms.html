Classical Algorithms: Decision Trees to XGBoost

Ensemble Learning, Random Forests, and Boosting Powerhouses

1. Decision Trees

The foundation of tree-based learning. It works by recursively splitting data based on feature thresholds that maximize Information Gain or minimize Gini Impurity.

2. Ensemble Learning: Wisdom of the Crowd

Instead of relying on one "Smart" tree, we combine many "Weak" models to create a "Strong" one. This leads to two main strategies:

Bagging (Bootstrap Aggregating)

Builds multiple independent trees in parallel and averages their results. Example: Random Forest.

Benefit: Reduces Variance (Overfitting).

Boosting

Builds trees sequentially. Each new tree focuses on correcting the errors of the previous ones. Example: XGBoost, LightGBM, CatBoost.

Benefit: Reduces Bias and Variance (High Accuracy).

3. XGBoost: The Kaggle King

eXtreme Gradient Boosting is the most popular algorithm for tabular data. It differs from standard GBM by using a more rigorous mathematical framework for tree growth.

XGBoost Internals (The "Extreme" Part)

XGBoost minimizes a regularized objective function using a second-order Taylor expansion of the loss function.

Obj = Σ L(y_i, ŷ_i^{(t-1)} + f_t(x_i)) + Ω(f_t)
  • Second-Order Approximation: Unlike basic GBM which uses only gradients, XGBoost uses both Gradients (g) and Hessians (h) (second derivatives), leading to faster convergence.
  • Regularization (Ω): Penalizes tree complexity (number of leaves and leaf weights) to prevent overfitting.
  • Optimal Split: XGBoost calculates a "Gain" score for every potential split based on gradients and hessians, choosing the one that maximizes improvement.

Key performance features:

  • Parallel Processing: Unlike standard GBM, it builds tree structures in parallel by sorting data beforehand.
  • Built-in Cross-Validation: Allows pruning and stopping at the exact moment overfitting begins.
  • Sparsity Awareness: Handles missing values by learning a "default direction" for them during training.

4. Support Vector Machines (SVM)

Finds the "Maximum Margin Hyperplane" that separates two classes. It uses the Kernel Trick to map data into higher dimensions where they become linearly separable.
