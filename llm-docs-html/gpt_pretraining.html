GPT Pretraining & Scaling Laws

Next-Token Prediction, Cross-Entropy Loss, and the Chinchilla ratio

The Unreasonable Effectiveness of Next-Token Prediction

Every breakthrough in modern AI—from writing code to passing the Bar Exam—stems from a shockingly simple pretraining objective: Next-Token Prediction (also known as Causal Language Modeling or Autoregressive Generation).

Given the context "The sky is", the model must output a probability distribution over its entire vocabulary (e.g., 50,000 words), assigning the highest probability to the actual target token (e.g., "blue").

Why is this so powerful? To perfectly predict the next word across a trillion tokens of internet text, the model is physically forced to learn grammar, facts, reasoning, logic, and physics. To predict the next word in a physics textbook problem, it has to learn how to solve the problem.

The Mathematics of Learning (Cross-Entropy Loss)

How do we actually update the model's weights? By measuring how "wrong" its probability distribution was compared to reality.

Loss = - Σ (y_true * log(y_pred))

In standard pretraining, y_true is a one-hot encoded vector: the target word ("blue") has a value of 1.0, and every other word is exactly 0.0.

If the model assigned a 99% probability (0.99) to "blue", the log of 0.99 is nearly 0, so the loss is nearly 0 (no gradients). If the model only assigned a 1% probability (0.01) to "blue", the log of 0.01 is a massive negative number, generating a massive Loss, which backpropagates a massive gradient to violently adjust the weights.

The Chinchilla Scaling Laws (2022)

For years, companies built larger and larger models (GPT-3 had 175 Billion parameters) but trained them on very little data (300 Billion tokens). DeepMind's "Chinchilla" paper mathematically proved this was a massive waste of resources.

The Compute-Optimal Ratio: 20x

For every 1 parameter in a model, you should train it on exactly 20 tokens of data. If you have a budget to train a massive 100B model on 300B tokens (a 3x ratio), you will get a significantly smarter, better model by training a tiny 15B model on those same 300B tokens.

The LLaMA Revolution: Meta took this to the extreme with LLaMA (1, 2, and 3). Instead of the 20x Chinchilla optimal limit, they trained small 7B and 8B parameter models on 15 Trillion tokens (a 2000x ratio!). This created "over-trained" models that are small enough to run on a laptop but completely crush massive older models like GPT-3.

Emergent Abilities

As you scale either the parameter count or the token count, researchers noticed Emergent Abilities. A model with 1B parameters might completely fail at arithmetic. At 5B, it still fails. At 10B, suddenly and unpredictably, it masters 3-digit arithmetic. The model undergoes a phase transition, suddenly wiring together the intermediate subnetworks required to solve complex reasoning problems.
