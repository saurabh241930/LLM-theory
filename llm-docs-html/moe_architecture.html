Mixture of Experts (MoE)

Sparse Activation and the death of monolithic networks

The Mathematics of compute bounds

If you train a 100-Billion parameter dense model like GPT-3, every single one of those 100 billion parameters must be loaded into VRAM and mathematically multiplied against the input token. Whether the user asks "What is 2+2" or writes a 4,000 line Python OS kernel, the exact same computational price is paid. This is incredibly inefficient.

How the Router works:
The Router is a tiny, incredibly fast feed-forward network. It computes the dot product of the incoming token against its routing weights, generating a probability distribution via Softmax.

Instead of activating the actual expert networks (which hold the massive 100B weights), it forces Top-K sparsity (usually Top-1 or Top-2).
If Top-1 routing is enabled, 99% probability is funneled entirely to Expert 1, and the other 7 Experts are completely bypassed. The GPU literally skips doing the math for 85 Billion parameters.

This is why Mixtral 8x7B (56 Billion total parameters) runs at the inference speed of a tiny 14B model. The remaining 42 Billion weights sit completely dormant in RAM during that token's inference cycle.
