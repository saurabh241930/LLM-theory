Production Engineering at Scale: SLOs, Reliability, and Observability

1. The DevOps Philosophy

DevOps is not a role, but a cultural and professional movement that stresses communication, collaboration, and integration between software developers (Dev) and IT professionals (Ops).

C.A.L.M.S Framework

  • Culture: Break down silos.
  • Automation: Eliminate toil.
  • Lean: Small batches, fast feedback.
  • Measurement: Data-driven decisions.
  • Sharing: Open communication.

SRE vs DevOps

"class SRE implements DevOps"

While DevOps is the philosophy, Site Reliability Engineering (SRE) is the concrete implementation. SREs use engineering principles to maintain system availability and performance.

2. Reliability Engineering (The Math)

SREs manage reliability using three core metrics: SLIs, SLOs, and SLAs.

SLI vs SLO vs SLA

Service Level Indicator (SLI): A specific metric (e.g., Latency, Error Rate).
Service Level Objective (SLO): The target value for an SLI (e.g., Latency < 200ms for 99.9% of requests).
Service Level Agreement (SLA): The contract with the customer (e.g., If uptime < 99.9%, we refund you money).

Error Budget = (1 - SLO) * Total Requests
Example: If SLO is 99.9% and you have 1M requests, your budget for errors is 1,000 requests.

3. The Three Pillars of Observability

Monitoring tells you what is wrong. Observability tells you why.

Metrics: Numerical data points aggregated over time.

Example: CPU usage 85%, HTTP 200 count 1500/min. Great for alerting and dashboards.
