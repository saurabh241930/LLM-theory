Advanced PEFT Types

Exploring DoRA, GaLore, IA3, and the future of parameter efficiency

1. DoRA: Weight-Decomposed LoRA

Problem: Standard LoRA couples magnitude and direction updates together. If you change the direction of a weight, you inadvertently change its magnitude.

Solution: DoRA decomposes the weight matrix into a magnitude vector and a direction matrix. It then applies LoRA updates only to the direction matrix. This makes fine-tuning closer to full fine-tuning in terms of learning capability.

📈 Result: DoRA consistently outperforms LoRA on reasoning tasks (GSM8K) while maintaining the same parameter count.

2. GaLore: Gradient Low-Rank Projection

Difference: GaLore is not an "adapter." It allows Full Fine-Tuning of the model but reduces memory by projecting the gradients into a low-rank subspace.

  • Pros: Updates 100% of weights, no extra inference latency.
  • Cons: More complex setup than LoRA.
# GaLore initialization in training config
optim_params = {"rank": 128, "update_proj_gap": 200}

3. IA3: Infused Adapter by Inhibiting and Amplifying Inner Activations

Mechanism: Instead of adding matrices, IA3 uses element-wise scaling of the internal activations (hidden states). It multiplies the key, value, and feed-forward activations by small learned vectors.

Why use it? It is even more efficient than LoRA, often requiring only 0.01% of parameters to be trainable.
