1. Summary

Despite its name, Logistic Regression is a classification algorithm. It takes a linear combination of features and passes it through the Sigmoid function to squash the output between 0 and 1. This output is interpreted as the probability of a data point belonging to a certain class.

2. Visual Flow

3. Mathematics

σ(z) = 1 / (1 + e-z)

Log-Loss: Penalizes confident wrong predictions exponentially.

Loss = -[y log(y_hat) + (1-y) log(1-y_hat)]

5. Scenarios & Use Cases

✅ Use Cases

  • Email Spam Detection (Spam vs. Not Spam).
  • Churn Prediction (Will the customer leave?).
  • Medical Diagnosis (Disease vs. Healthy).

⚠️ Caveats

  • Requires labels to be independent.
  • Assumes linear decision boundary between classes.
  • Vulnerable to Multicollinearity (features being highly correlated).

5. Real-World Case Study: Banking Loan Approval

The Problem: A commercial bank receives 10,000 credit card applications an hour and needs to decide "Approve" or "Reject".

The Solution: Logistic Regression outputs a probability score between 0 and 1.

  • Features: Credit Score, Debt-to-Income ratio, Employment length.
  • Result: If P(Default) > 0.2, the application is automatically flagged for review.

Why Logistic Regression? In finance, models must be Regulatorily Compliant. Logistic regression provides a mathematical "reason" (Log-odds) for every rejection, satisfying transparency laws.

6. Implementation

from sklearn.linear_model import LogisticRegression
from sklearn.datasets import make_classification

# 1. Synthetic Classification Data
X, y = make_classification(n_samples=100, n_features=2, random_state=42)

# 2. Train Model
clf = LogisticRegression()
clf.fit(X, y)

# 3. Predict Probability
probs = clf.predict_proba(X[:2])
print(f"Probabilities: {probs}")
