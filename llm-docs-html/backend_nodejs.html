V8 Engine, Libuv & The Real Event Loop

1. Architecture: V8 & Libuv

Node.js is a runtime that pairs Google's V8 Engine (the JS parser) with Libuv (the C library that handles the event loop and thread pool).

graph LR JS[JavaScript Code] --> V8[V8 Engine] V8 --> NodeAPI[Node.js Bindings] NodeAPI --> Libuv[Libuv Event Loop] Libuv --> OS[Operating System Kernal]

2. The 6 Phases of the Node.js Event Loop

Q: Can you name the primary phases of the Libuv loop?

  1. Timers: Executes callbacks from setTimeout and setInterval.
  2. Pending Callbacks: Executes I/O callbacks deferred to the next iteration.
  3. Idle, Prepare: Internal use only.
  4. Poll: Retrieves new I/O events; executes I/O-related callbacks.
  5. Check: Executes setImmediate() callbacks.
  6. Close Callbacks: Executes closing handles like socket.on('close', ...).
Senior Signal: process.nextTick() is NOT part of the event loop. It's part of the **Microtask Queue**, which is processed immediately after the current operation finishes, before the next phase starts.

3. Worker Threads vs Clustering

Q: When do you use Worker Threads vs. the Cluster Module?

Use Clusters to scale to multiple CPU cores (horizontal scaling on one machine); use Worker Threads for heavy CPU-bound tasks in a single process (vertical scaling).

Clusters create separate processes with their own memory; Worker Threads share memory using ArrayBuffer.

4. Node.js Streams & Backpressure

Q: What is Backpressure in Node.js streams?

When a data source (Readable) sends data faster than the destination (Writable) can process it.

Fix: Streams signal the source to pause until the buffer is drained. Always use pipeline() instead of .pipe() for better error handling.

Q1: What is the primary difference between `process.nextTick()` and `setImmediate()`?

nextTick fires immediately after the current operation and before the event loop continues. setImmediate fires in the 'check' phase of the next loop iteration. nextTick can starve I/O if called recursively.

Q2: How does Node.js handle heavy CPU tasks without blocking the main thread?

By using Worker Threads (worker_threads module) for true parallelism or the Cluster module for process-level scaling. For simple async tasks, Libuv's thread pool is used.

Q3: Explain 'Backpressure' in Node.js Streams.

Backpressure occurs when the data source (Readable) is faster than the destination (Writable). The writable stream returns `false` on `.write()`, signaling the readable stream to pause until 'drain' is emitted.

Q4: What are 'Async Hooks' in Node.js?

The `async_hooks` module provides an API to track the lifetime of asynchronous resources (like promises or timers). Useful for distributed tracing and context propagation.

Q5: Difference between `module.exports` and `exports`?

`module.exports` is the actual object returned by `require()`. `exports` is just a reference to `module.exports`. If you reassign `exports`, you break the reference.

Q6: How would you debug a Memory Leak in Node.js?

Use `node --inspect`, take heap snapshots via Chrome DevTools, and look for 'Detached DOM nodes' equivalent in Node (objects that should be GC'd but are held by global roots or closures).

Q7: Describe the purpose of the 'Libuv' Thread Pool size.

By default, Libuv has 4 threads for handling FS, DNS, and Crypto tasks. Increasing `UV_THREADPOOL_SIZE` can improve performance for high-volume disk/crypto operations.

Q8: What is 'Zombie Process' and how to prevent it in Cluster module?

A zombie process occurs when a child process terminates but remains in the process table because the parent hasn't 'waited' for it. Node's `cluster` module handles this automatically, but manually spawned processes need `.on('exit')` cleanup.

Q9: How do you secure a Node.js API against ReDoS (Regular Expression Denial of Service)?

Avoid 'evil regex' with catastrophic backtracking (nested quantifiers like `(a+)+$`). Use `safe-regex` check or timeout-based regex execution.

Q10: Explain the 'Stream.pipeline()' vs 'Stream.pipe()'.

`pipeline()` properly handles errors by forwarding them to a callback and cleaning up all streams in the chain. `.pipe()` doesn't necessarily close all streams on error.

Q11: What is the role of `Buffer.allocUnsafe()`?

It allocates memory without zero-filling it. It is faster but can contain sensitive data from previous allocations. Use only when you are sure to overwrite the entire buffer.

Q12: How does the `cluster` module share the port?

The primary process listens on the port and hands off handles to worker processes using Round-Robin (except on Windows).

Q13: What is 'Hardening' a Node.js production app?

Using Helmet for headers, setting `NODE_ENV=production`, disabling `X-Powered-By`, limiting payload size, and using a non-root user in Docker.

Q14: Explain 'CJS' (CommonJS) vs 'ESM' (ES Modules) in Node.

CJS is synchronous and uses `require()`. ESM is asynchronous, supports top-level await, and use `import`. ESM is the future-proof standard.

Q15: How to handle Uncaught Exceptions and Unhandled Rejections?

Listen to `process.on('uncaughtException')` and `unhandledRejection`. Always log the error and perform a graceful shutdown (`process.exit(1)`) as the app state is now unreliable.

Q16: How do you achieve 100% test coverage for async code?

Use `null` injection for mocks, simulate network timeouts, and ensure every `catch` block is triggered in unit tests via forced errors.

Q17: What is 'Tracing' vs 'Logging' in a distributed Node ecosystem?

Logging records discrete events. Tracing (OpenTelemetry) tracks a single request across multiple services using a TraceID/SpanID.

Q18: How does `crypto.pbkdf2` work?

It's a key derivation function with a salt and many iterations to prevent brute-force attacks on passwords. It runs on the Libuv thread pool.

Q19: Explain the 'Reactor Pattern' in Node.js.

It's the design pattern where a service handler demultiplexes incoming requests and dispatches them synchronously to associated request handlers.

Q20: What is 'Garbage Collection Strategy' in V8?

V8 uses a generational GC (Scavenger for young generation, Mark-Sweep-Compact for old generation). It uses a 'Stop-the-world' approach but optimizes for short pauses.

Q21: How to implement Rate Limiting without Redis?

Use memory-stores like `lru-cache` or `express-rate-limit` built-in memory store. Risk: it resets on restart and doesn't share state between cluster workers.

Q22: Difference between `spawn()`, `exec()`, and `fork()`?

`spawn` streams data (best for large data). `exec` buffers output (best for small commands). `fork` creates a new Node process with an IPC channel.

Q23: How do you measure 'Event Loop Lag'?

By measuring the delay between a `setTimeout(..., 0)` callback and the current time. High lag indicates a CPU-bottlenecked main thread.

Q24: What is the 'Buffer' size limit?

In 64-bit Node, it's roughly 2GB-4GB depending on the version. Large buffers can lead to 'Array buffer allocation failed' errors.

Q25: Explain 'Hot Path' optimization.

Identifying functions that run most frequently and optimizing them (e.g., avoiding unnecessary object allocations or using bitwise ops where appropriate).

Q26: How to handle 'Circular Dependencies' in CJS?

CJS allows it by returning an incomplete `exports` object. It can lead to 'undefined' if you access properties too early. Best to refactor.

Q27: What is `AbortController` in modern Node?

A standard way to cancel asynchronous tasks like `fetch` or file reads. It emits an 'abort' event.

Q28: Importance of `package-lock.json`?

Ensures deterministic builds. It locks the exact version of every nested dependency so that 'npm install' yields the same 'node_modules' on all machines.

Q29: How to protect against 'NoSQL Injection' in MongoDB/Node?

Sanitize inputs. Never pass raw user objects to queries like `db.collection.find(req.body)`. Use Mongoose schemas.

Q30: Explain 'Graceful Shutdown'.

Closing server to new connections, finishing pending requests, closing DB pools, and finally `process.exit(0)` after a timeout.

Q31: What is the 'REPL' in Node?

Read-Eval-Print Loop. An interactive shell for testing JS code snippets directly in the terminal.

Q32: Role of 'N-API' / 'Node-API'?

A stable C API for writing native add-ons that is independent of the V8 version, preventing breakage on Node upgrades.

Q33: How to optimize 'JSON.parse' for huge payloads?

Use `JSONStream` or `oboe.js` to parse objects incrementally rather than loading the entire string into memory.

Q34: What is 'Dead Letter Queue' in Node task processing?

A queue for messages that failed processing multiple times. Prevents 'poison pills' from blocking the main queue.

Q35: Explain 'Dependency Injection' in Node.js.

Passing dependencies (DB, services) into constructors or functions instead of requiring them globally. Eases testing and modularity.

Q36: How to share memory between Workers?

Use `SharedArrayBuffer` and `Atomics` for thread-safe operations on shared memory.

Q37: What is 'SemVer'?

Semantic Versioning: MAJOR.MINOR.PATCH. ^1.2.3 allows minor/patch updates; ~1.2.3 allows only patch updates.

Q38: How to prevent 'Callback Hell'?

Use Promises, then chain them; or use `async/await`. Use `util.promisify` to convert older callback-based APIs.

Q39: What is the 'Global Object' in Node?

`global`. It's the equivalent of `window` in the browser. Avoid polluting it.

Q40: How to profile a Node.js app performance?

Use `node --prof`, `clinic.js`, or `0x` (flamegraphs) to find CPU bottlenecks.

Q41: Difference between `fs.readFile` and `fs.createReadStream`?

`readFile` loads the whole file into RAM. `createReadStream` reads in chunks (low memory footprint).

Q42: What is 'Event Orientation' vs 'Object Orientation' in Node?

Node is heavily 'Even-Oriented' using EventEmitter. OO is used for structure, but communication is often via events.

Q43: How to use `pnpm` vs `npm`?

`pnpm` uses hard links and symlinks to save disk space and is generally faster due to its content-addressable storage.

Q44: What is 'Serverless Node.js' cold start?

The delay when a function is invoked for the first time in a while. Reduced by minimizing package size and dependencies.

Q45: How to handle 'Large Integer' precision in JS (Node)?

Use `BigInt` (e.g., `100n`). Numbers in JS are 64-bit floats and lose precision above ~9 quadrillion.

Q46: Role of `.npmrc`?

Configuration file for npm. Used for setting registry URLs, auth tokens for private packages, and installation defaults.

Q47: What is 'Middle-ware' pattern?

A series of functions that process a request/response object sequentially. Used in Express/Koa for auth, logging, etc.

Q48: How to implement 'Circuit Breaker' in Node?

Use libraries like `opossum`. It stops making calls to a failing downstream service to allow it to recover.

Q49: What is 'SSR' (Server Side Rendering) with Node?

Rendering the initial HTML on the server (using React/Vue) before sending it to the client for faster FCP (First Contentful Paint).

Q50: Why is 'Node.js' popular for Microservices?

Lightweight, fast startup, easy to dockerize, and built-in support for async I/O and streaming.
