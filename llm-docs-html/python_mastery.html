Python Mastery for AI Engineers

Internals, Concurrency, Meta-programming, and Production Patterns

Q1: Explain the difference between Generators and Iterators.

All generators are iterators, but not all iterators are generators. Generators use `yield` and maintain state automatically.

Core Points

  • Iterator: Any object implementing `__iter__()` and `__next__()`.
  • Generator: A function that uses `yield` to return a stream of values lazily.
  • Benefit: Memory efficiency (O(1) memory for large datasets) and readability.
Senior Signal: Generators are "pausable functions." When `yield` is hit, the local state (variables, instruction pointer) is saved, not destroyed. This is the foundation for `asyncio` (before the native `async` keyword).

Q2: How does Python's AsyncIO (Event Loop) differ from Multi-threading?

Multi-threading is for I/O-bound tasks using OS-level context switching; AsyncIO is cooperative multitasking on a single thread.

Core Points

  • Threads: Preemptive multitasking. OS switches between threads. High memory overhead. Risk of race conditions.
  • AsyncIO: Cooperative multitasking. Functions "yield" control back to the event loop. Low overhead. Ideal for 10k+ concurrent connections (e.g., FastAPI).
Trap: `asyncio` does not make CPU-bound tasks faster. Blocking a coroutine with `time.sleep()` blocks the entire event loop. Use `run_in_executor` for blocking calls.

Q3: What is the Global Interpreter Lock (GIL) and why does it exist?

A mutex that allows only one thread to execute Python bytecode at a time, ensuring thread-safety for CPython's memory management.

Core Points

  • Why: Protects reference counting from race conditions.
  • Impact: Makes multi-threading useless for CPU-heavy tasks in Python.
  • Workaround: Use `multiprocessing` to bypass the GIL or use libraries like NumPy/PyTorch which release the GIL during heavy C-level computation.

Q4: Explain Decorators and how to preserve function metadata.

Decorators wrap functions to add behavior. Use `functools.wraps` to preserve the original function's name and docstrings.
from functools import wraps def logger(func): @wraps(func) def wrapper(*args, **kwargs): print(f"Calling {func.__name__}") return func(*args, **kwargs) return wrapper

Q5: Why use Pydantic for AI applications?

Strong type validation and serialization at runtime, essential for parsing LLM JSON outputs and API contracts.

Core Points

  • Validation: Automatically converts and validates types (e.g., string to int).
  • Schema: Generates JSON maps for Function Calling automatically.
  • Performance: Highly optimized (V2 is written in Rust).

Q6: Explain `__getattr__` vs `__getattribute__`.

`__getattribute__` is called for every attribute access; `__getattr__` is only called if the attribute is not found.
Senior Signal: Use `__getattr__` for lazy loading or proxy patterns. Be extremely careful with `__getattribute__` as it can easily cause infinite recursion (always use `super().__getattribute__`).

Q7: How do Metaclasses work?

A metaclass is a "class of a class." It defines how a class is constructed.

Classes are instances of `type`. By inheriting from `type`, you can intercept class creation (e.g., to enforce naming conventions or auto-register plugins).

Q8: List vs Tuple vs Set performance.

  • List: O(1) append, O(n) search. Over-allocates memory for growth.
  • Tuple: O(1) access. Immutable. Slightly faster/smaller than list due to fixed size.
  • Set: O(1) average case search (Hash table). Great for deduplication.

Q9: Explain Python's Memory Management (Reference Counting & GC).

Python uses reference counting primary; a generational Garbage Collector (GC) handles cyclic references.

Core Points

  • Ref Counting: Object is deleted as soon as its count hits zero.
  • Generational GC: Scans objects in three generations (0, 1, 2) to find and break loops like `a.x = b; b.x = a`.

Q10: What are Context Managers (`with` statement)?

Objects that define setup and teardown logic via `__enter__` and `__exit__`.

Use `@contextlib.contextmanager` for a generator-based approach. Essential for resource management (files, DB connections, locks).

Q11: `typing.Protocol` vs Abstract Base Classes (ABC).

ABC is nominal subtyping (must inherit); Protocol is structural subtyping (duck-typing for static analysis).
Senior Signal: Use `Protocol` when you want to define a type based on "what it can do" rather than "who its parents are." It's Python's version of Go interfaces.

Q12: How to profile a Python script?

Use `cProfile` for function call stats, `line_profiler` for line-by-line time, and `memray` or `tracemalloc` for memory.
python -m cProfile -s time my_script.py

Q13: Difference between `__str__` and `__repr__`.

`__str__` is for users (pretty); `__repr__` is for developers (unambiguous, ideally should reproduce the object).

Q14: Shallow vs Deep Copy.

Shallow copy creates a new container but references the same nested objects; Deep copy recursively copies everything.

Q15: What is the `__slots__` attribute?

Prevents the creation of `__dict__` for each instance, saving significant memory for objects with many instances.
Trap: `__slots__` also prevents adding new attributes dynamically at runtime.

Q16: How do you handle circular imports?

Move imports inside functions, use string-based type hints, or refactor shared logic into a third module.

Q17: Explain `collections.defaultdict` vs `dict.get()`.

`defaultdict` auto-creates keys on access; `dict.get()` returns a value or default without modifying the dict.

Q18: What is a Closure?

A nested function that remembers the environment in which it was created, even after the outer function has finished.

Q19: `is` vs `==`.

`is` checks identity (memory address); `==` checks equality (values).

Q20: What are "Dunder" (Double-Under) methods?

Special methods like `__init__`, `__call__`, `__len__` that allow objects to hook into Python's built-in syntax (operators, casting, etc.).

Q21: `multiprocessing` vs `threading` — which one for AI?

`multiprocessing` for CPU-bound tasks (Parallelism); `threading` for I/O-bound tasks (Concurrency).

Core Points

  • Multiprocessing: Creates separate memory spaces. Bypasses GIL. Use for heavy data processing / tokenization.
  • Threading: Shared memory space. Limited by GIL. Use for network requests or DB queries.

Q22: Explain `typing.Generic` and `typing.TypeVar`.

Used to create components that work with a variety of types while maintaining type safety.
from typing import TypeVar, Generic T = TypeVar('T') class Box(Generic[T]): def __init__(self, content: T): self.content = content

Q23: Difference between `Any` and `object` types.

`object` is the root of all types (strictly checked); `Any` tells the type checker to "trust me" and disable checking.

Q24: What does the `__call__` method do?

Allows an instance of a class to be called like a function.

Commonly used for stateful decorators or defining "Functors."

Q25: Explain `itertools.islice` and why it's useful.

Allows slicing an iterator without converting it to a list, preserving memory.
from itertools import islice # Get first 10 items from a massive generator for item in islice(massive_gen, 10): print(item)

Q26: What is `functools.lru_cache`?

A decorator that wraps a function with a Least Recently Used (LRU) cache, memoizing results for seen inputs.
Senior Signal: Essential for recursive algorithms or expensive idempotent API calls. In Python 3.9+, use `@cache` for an infinite LRU cache.

Q27: Pydantic `field_validator` vs `model_validator`.

`field_validator` checks individual fields; `model_validator` checks multiple fields together (cross-field validation).

Q28: How to use `subprocess.run` securely?

Always pass arguments as a list, never a single string with `shell=True`, to prevent shell injection.

Q29: Explain `typing.Annotated` and its use in FastAPI.

Adds metadata to types without changing the type itself.

FastAPI uses it to combine types with dependency injection: `db: Annotated[Session, Depends(get_db)]`.

Q30: What are "Soft Keywords" in Python (e.g., `_` in match)?

Keywords that only act as keywords in specific contexts (like `match` and `case` in Python 3.10), preserving backward compatibility for identifiers.

Q31: Difference between `dataclasses` and `NamedTuple`.

`dataclasses` are mutable by default and support default factory values; `NamedTuple` is immutable and has lower memory overhead.

Q32: What is "Monkey Patching"?

Modifying objects or modules at runtime. Useful for testing (mocking) but dangerous in production.

Q33: How does `super()` work in Multiple Inheritance?

It follows the Method Resolution Order (MRO), calculated using the C3 Linearization algorithm.
Senior Signal: Use `ClassName.mro()` to inspect the order. `super()` does NOT necessarily call the parent; it calls the next class in the MRO.

Q34: Explain the `nonlocal` keyword.

Allows a nested function to modify a variable in the nearest enclosing (non-global) scope.

Q35: What is `__init_subclass__` used for?

A simpler alternative to metaclasses for customizing class creation of subclasses.

Q36: `pytest` vs `unittest` — why one over the other?

`pytest` is the industry standard for its concise syntax, powerful fixtures, and huge plugin ecosystem; `unittest` is built-in but verbose.

Q37: Explain the Walrus Operator (`:=`).

Allows assignment of a value within an expression.
if (n := len(data)) > 10: print(f"Too long: {n} items")

Q38: Pattern Matching (`match` / `case`) in Python 3.10+.

A powerful way to destructure data and execute code paths based on its shape.
match command.split(): case ["quit"]: exit() case ["load", filename]: load(filename) case _: raise ValueError("Unknown")

Q39: What are Exception Groups (`ExceptionGroup`)?

Introduced in Python 3.11 to handle multiple concurrent errors, especially in `asyncio`.

Use `except*` to catch specific exceptions within a group.

Q40: Explain `contextlib.ExitStack`.

Allows managing a dynamic number of context managers in a single `with` block.
Senior Signal: Essential for opening a variable list of files or cleaning up many mocks in testing.

Q41: Difference between `asyncio.gather` and `TaskGroup`.

`TaskGroup` (Python 3.11+) is the modern, safer way to manage concurrent tasks; it ensures if one fails, all are cancelled.

Q42: What is a "Weak Reference" (`weakref`)?

A reference that doesn't prevent an object from being garbage collected. Useful for caching and observer patterns.

Q43: How does the `yield from` syntax work?

Delegates part of a generator's operations to another generator, flattening the structure.

Q44: `pip` vs `poetry` vs `uv`.

  • pip: Basic installer. No built-in lockfiles.
  • Poetry: High-level tool for dependency resolution and packaging. Reliable lockfiles.
  • uv: An extremely fast pip replacement written in Rust. Currently the performant choice for CI/CD.

Q45: Explain Python's Integer Caching (-5 to 256).

Small integers are pre-allocated in memory. `a = 5; b = 5; a is b` is True, but for 257 it is False.

Q46: What is a "Static Method" vs "Class Method"?

`@classmethod` receives the class (`cls`) as the first argument; `@staticmethod` receives nothing and is just a namespace organizer.

Q47: How to handle Byte strings vs Unicode in Python?

Always `decode()` bytes into strings on input and `encode()` strings into bytes on output (the "Unicode Sandwich").

Q48: What is `sys.path`?

A list of strings that specifies the search path for modules. Empty string at index 0 refers to the current directory.

Q49: Explain Type Hinting `Protocol` with `runtime_checkable`.

Allows checking if an object implements a `Protocol` at runtime using `isinstance()`.

Q50: Why is Pydantic so popular in Gen-AI pipelines?

It makes "Structured Output" from LLMs predictable. By feeding a Pydantic model's JSON schema to an LLM, we can enforce specific output formats with 99%+ reliability.
Senior Signal: Libraries like Instructor or LangChain's OutputParsers wrap Pydantic to handle retries and validation errors automatically when an LLM hallucinates a bad field.

Pythonic Coding & Logic

Q51: Explain List Comprehensions vs `map()`.

List comprehensions are generally more readable and faster in Python; `map()` is primarily useful when applying an existing function to an iterable.
# List Comp (Preferred) squares = [x**2 for x in range(10) if x % 2 == 0] # Map (Verbose) squares = list(map(lambda x: x**2, filter(lambda x: x % 2 == 0, range(10))))

Q52: Advanced Dictionary Manipulation — Merging and Filtering.

In Python 3.9+, use the `|` operator for merging. Use dict comprehensions for filtering.
dict_a = {"a": 1, "b": 2} dict_b = {"b": 3, "c": 4} # Merge (dict_b wins on conflict) merged = dict_a | dict_b # Filtering (only values > 2) filtered = {k: v for k, v in merged.items() if v > 2}

Q53: How to flatten a Nested List efficiently?

Nested list comprehensions or `itertools.chain`.
nested = [[1, 2], [3, 4], [5]] flattened = [item for sublist in nested for item in sublist]

Q54: Dictionary Sorting — by Key and by Value.

Use the `sorted()` function with a `lambda` key.
scores = {"Alice": 90, "Bob": 80, "Charlie": 95} # Sort by value (Descending) sorted_scores = dict(sorted(scores.items(), key=lambda item: item[1], reverse=True))

Q55: Common Trap: Mutable Default Arguments.

Using a list or dict as a default argument will lead to shared state across function calls.
def add_item(item, list_=[]): # WRONG: list_ is shared list_.append(item) return list_ def add_item_correct(item, list_=None): # CORRECT if list_ is None: list_ = [] list_.append(item) return list_
