1. Supervised Learning: The Starting Point

Before jumping into LLMs, we must understand the core of prediction: Supervised Learning. This is where a model learns a mapping from inputs (X) to outputs (y) based on labeled examples.

2. Linear Regression (Predicting Numbers)

Linear regression is the simplest algorithm, often used for predicting continuous values (e.g., house prices, sales revenue).

ŷ = β₀ + β₁x₁ + β₂x₂ + ... + ε

In our RSA Sales QA case study, linear regression could be used to predict the length of a call based on the complexity of the inquiry.

3. Logistic Regression (Predicting Classes)

Despite the name, this is a Classification algorithm. It predicts the probability that an input belongs to a certain category (0 or 1).

The Sigmoid Function: Maps any real value into a range between 0 and 1.
σ(z) = 1 / (1 + e⁻ᶻ)

Use Case: Predicting if a sale will be "Successful" or "Unsuccessful" based on historical customer interaction data.

4. The Bias-Variance Tradeoff

This is the most critical concept in all of Machine Learning. It explains why models fail to generalize.

Concept Problem State Solution
High Bias Underfitting Too simple, misses patterns Add complexity / features
High Variance Overfitting Too complex, learns noise Regularization / Simplify

Mathematical Decomposition

The total error of a model can be broken down into three components:

Err(x) = E[(y - ŷ)²] = Bias[ŷ]² + Var[ŷ] + σ²
  • Bias²: (E[ŷ] - y)² — Difference between the average prediction and the true value.
  • Variance: E[(ŷ - E[ŷ])²] — How much the predictions for a given point vary between different realizations of the model.
  • Irreducible Error (σ²): The noise in the data itself that no model can capture.

5. Regularization: Taming Variance (L1 & L2)

Regularization adds a penalty term to the loss function to prevent the model from learning noise (overfitting).

L1 Regularization (Lasso)

Adds the absolute value of weights to the loss.
Loss = Error + λ Σ|w|

Property: Can shrink some coefficients to exactly zero. Useful for feature selection when data has many redundant features.

L2 Regularization (Ridge)

Adds the squared value of weights to the loss.
Loss = Error + λ Σw²

Property: Shrinks weights to be small but never exactly zero. Prevents any single feature from dominating the prediction (weight decay).

ElasticNet

A hybrid approach that combines both L1 and L2 penalties. Useful when there are multiple correlated features.

Loss = Error + λ₁ Σ|w| + λ₂ Σw²
Feature L1 (Lasso) L2 (Ridge)
Penalty |w| (L1 Norm) w² (L2 Norm)
Weight Zeroing Yes (Sparse models) No (Small weights)
Best for... Feature Selection Preventing Overfitting

6. How do we know it works? (Evaluation Metrics)

Accuracy is often misleading (e.g., in imbalanced datasets like fraud detection). We need better measurements.

The Confusion Matrix

Fundamental tool for binary classification:

  • TP / TN: Correct predictions.
  • FP (Type I Error): False Alarm.
  • FN (Type II Error): Failure to detect.
Metric Formula When to prioritize?
Precision TP / (TP + FP) When cost of FP is high (e.g., spam detection).
Recall TP / (TP + FN) When cost of FN is high (e.g., cancer detection).
F1 Score 2 * (P * R) / (P + R) Harmonic mean—best for imbalanced datasets.
ROC-AUC Area under Curve Measures model's ability to distinguish between classes.

7. Cross-Validation (K-Fold)

Instead of a single train/test split, we split the data into K folds. We train the model K times, each time using a different fold as the validation set.

graph LR subgraph Iteration 1 V1[Val] --- T1[Train] --- T2[Train] --- T3[Train] end subgraph Iteration 2 T4[Train] --- V2[Val] --- T5[Train] --- T6[Train] end

Benefit: Provides a more robust estimate of model performance and ensures we use our entire dataset for verification.
