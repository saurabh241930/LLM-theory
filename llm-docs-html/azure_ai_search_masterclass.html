3. Semantic Ranker: The L2 Secret

Azure's Semantic Ranker is a secondary ranking pass that uses Bing's deep learning models to re-order the Top 50 results. It doesn't just look for "keywords"; it understands Intent.

The Dual Ranking Pattern

  • Pass 1 (Hybrid): BM25 (Keyword) + HNSW (Vector) combined via RRF. Fast, finds candidates.
  • Pass 2 (Semantic): Cross-encoder re-ranks the hybrid results. Slower, but highest precision.

4. Implementation & OData Filtering

Unlike simple Vector DBs, Azure AI Search supports OData v4 complex filtering, allowing you to combine semantic similarity with structured business rules.

# Hybrid Search with Semantic Ranking (Python SDK)
from azure.search.documents import SearchClient
from azure.search.documents.models import VectorQuery, SemanticQuery

client = SearchClient(endpoint, index_name, credential)

results = client.search(
    search_text="How to claim RSA benefits?",
    vector_queries=[VectorQuery(vector=query_emb, k_nearest_neighbors=50, fields="embedding")],
    # The Magic: Semantic Ranking
    query_type="semantic",
    semantic_configuration_name="my-config",
    # OData Filter: Security & Metadata
    filter="category eq 'insurance' and user_role ge 3"
)
