Author: Antigravity AI | Topic: Imbalanced Classification | Dataset: Kaggle Credit Card Fraud

1. Project Description

Identifying fraudulent credit card transactions is a critical challenge for financial institutions. The dataset contains transactions made by credit cards in September 2013 by European cardholders.

The Challenge: The dataset is highly imbalanced, where the positive class (frauds) account for only 0.172% of all transactions. Accuracy is not a useful metric here; we must focus on Precision, Recall, and AUPRC (Area Under Precision-Recall Curve).

3. Data Cleaning & Preprocessing

We scale the Amount feature using RobustScaler (less sensitive to outliers) and drop the Time feature as it doesn't provide predictive value in this baseline.

3. Feature Engineering & Sampling

Since the classes are imbalanced, we apply SMOTE (Synthetic Minority Over-sampling Technique) to balance the training data.

In [4]:
# Rescaling Amount (not PCA-ed)
