DevOps & SRE Interview Q&A

50+ High-Impact Questions for Senior Infrastructure Roles

1. DevOps & SRE Foundations

CULTURAL What is "Toil," and how do you manage it?
Toil is work that is manual, repetitive, automatable, tactical, and devoid of enduring value. SREs aim to cap toil at 50% of their time, spending the other 50% on project work (engineering) to improve the system so that toil is reduced.
SRE You have an Error Budget of 0. What happens?
When the Error Budget is exhausted, all new feature releases are typically frozen until the service reliability is restored (back to SLO levels). Only "security" or "reliability-improving" changes are allowed.

2. Docker & Container Internals

DOCKER Difference between CMD and ENTRYPOINT?
ENTRYPOINT defines the executable that runs when the container starts. It's difficult to override.
CMD defines the default arguments for the ENTRYPOINT. It is easily overridden by command line arguments.
SECURITY How do you minimize the size of a Docker image?
Use lightweight base images (Alpine/Distroless), utilize multi-stage builds, combine RUN instructions to reduce layers, and use .dockerignore to skip unnecessary files.

3. Kubernetes Orchestration

K8S What is a Sidecar Pod, and why use it?
A Sidecar is a secondary container running in the same Pod as the main application container. It typically handles cross-cutting concerns like logging (Filebeat), monitoring (Prometheus exporter), or network proxying (Envoy/Istio).
TROUBLESHOOTING Explain "CrashLoopBackOff" and how to debug it.
CrashLoopBackOff occurs when a pod starts, crashes, and K8s tries to restart it repeatedly with an increasing delay. Debugging: Use `kubectl logs `, `kubectl describe pod `, and check for OOMKills, misconfigurations, or missing environment variables.

4. CI/CD & Infrastructure as Code

GITOPS What is GitOps, and how does it differ from traditional CI/CD?
GitOps is a practice where Git is the "single source of truth" for infrastructure. A controller (like ArgoCD) continuously monitors the Git repo and automatically updates the cluster state to match the repo, preventing "config drift."
